### **Function 1: deadband_filter**

**DeepSeek-chat output:**
```python
def deadband_filter(values, lower, upper):
    import numpy as np
    
    if not isinstance(values, (list, np.ndarray)):
        # single scalar
        if lower <= values <= upper:
            return 0
        elif values < lower:
            return values - lower
        else:
            return values - upper
    else:
        # array-like
        values = np.asarray(values)
        result = np.zeros_like(values)
        mask_below = values < lower
        mask_above = values > upper
        result[mask_below] = values[mask_below] - lower
        result[mask_above] = values[mask_above] - upper
        # values inside deadband already zero
        return result.tolist() if isinstance(values, list) else result
```

**Groq Llama-70b output:**
```python
def deadband_filter(values, lower, upper):
    if isinstance(values, (int, float)):
        if values < lower:
            return values - lower
        elif values > upper:
            return values - upper
        else:
            return 0
    else:
        result = []
        for v in values:
            if v < lower:
                result.append(v - lower)
            elif v > upper:
                result.append(v - upper)
            else:
                result.append(0)
        return result
```

**Comparison:**
- **Correctness**: Both handle scalar and list inputs. DeepSeek uses numpy for vectorization; Groq uses pure Python loops. DeepSeek's version incorrectly treats `values == upper` as inside deadband (good) but `values == lower` also inside (good). Groq's version treats equality (`v == lower` or `v == upper`) as inside deadband because `v < lower` and `v > upper` are false, so `else` returns 0. Both are correct for inclusive bounds.
- **Conciseness**: DeepSeek: 18 non‑comment lines. Groq: 14 lines. Groq is simpler.
- **Spec Adherence**: Both match signature. DeepSeek returns numpy array for numpy input, list for list input. Groq always returns list for non‑scalar. DeepSeek's behavior with numpy may be more flexible for fleet numeric work.

import re

def parse_fleet_log_line(line):
    pattern = r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}) \[([^\]]+)\] (\w+): (.+)$'
    match = re.match(pattern, line)
    if match:
        return {
            'timestamp': match.group(1),
            'agent': match.group(2),
            'level': match.group(3),
            'message': match.group(4)
        }
    return None

### **Function 3: constraint_snap**

**Prompt:**
```
Write a Python function `constraint_snap(point, line_start, line_end, tolerance)`.

- point, line_start, line_end are (x, y) tuples.
- If the shortest distance from point to the line segment is <= tolerance, snap the point to the nearest point on the segment.
- Otherwise return the original point.
- Use vector math; avoid unnecessary dependencies.

Provide only the function code.
```

**DeepSeek-chat output:**
```python
import math

def constraint_snap(point, line_start, line_end, tolerance):
    px, py = point
    x1, y1 = line_start
    x2, y2 = line_end
    
    # Vector from start to end
    dx = x2 - x1
    dy = y2 - y1
    seg_len_sq = dx*dx + dy*dy
    
    if seg_len_sq == 0:
        # line segment is a point
        dist = math.hypot(px - x1, py - y1)
        return (x1, y1) if dist <= tolerance else point
    
    # Projection parameter t
    t = ((px - x1) * dx + (py - y1) * dy) / seg_len_sq
    t = max(0, min(1, t))  # clamp to segment
    
    # Nearest point on segment
    nx = x1 + t * dx
    ny = y1 + t * dy
    
    # Distance to nearest point
    dist = math.hypot(px - nx, py - ny)
    return (nx, ny) if dist <= tolerance else point
```
