## **2. Function 1: `deadband_filter`**

**Prompt to both models:**
```
Write a Python function called `deadband_filter` that implements a deadband protocol for signal processing.

Function signature:
def deadband_filter(signal: list[float], threshold: float, apply_scaling: bool = False) -> list[float]:

Rules:
- P0 (negative space): If absolute value of a sample is <= threshold, set output to 0.0.
- P1 (safe channel): If absolute value > threshold, pass the sample through unchanged.
- P2 (optimize): Only if `apply_scaling` is True AND sample is outside deadband, scale the sample by (abs(sample) - threshold) / abs(sample) to "subtract" the deadband region.

Return the filtered list.

Include a brief docstring.
```

### **DeepSeek-chat Output:**
```python
def deadband_filter(signal: list[float], threshold: float, apply_scaling: bool = False) -> list[float]:
    """
    Apply deadband filtering to a signal.

    P0: Values within ±threshold of zero are set to zero.
    P1: Values outside deadband are passed through.
    P2: If apply_scaling is True, values outside deadband are scaled to remove deadband offset.

    Args:
        signal: Input signal as list of floats.
        threshold: Deadband threshold (positive).
        apply_scaling: Whether to apply P2 scaling.

    Returns:
        Filtered signal.
    """
    if threshold < 0:
        raise ValueError("Threshold must be non-negative")

    result = []
    for sample in signal:
        abs_sample = abs(sample)
        if abs_sample <= threshold:
            result.append(0.0)
        else:
            if apply_scaling:
                # Scale to remove deadband region
                sign = 1.0 if sample >= 0 else -1.0
                scaled = sign * (abs_sample - threshold)
                result.append(scaled)
            else:
                result.append(sample)
    return result
```

### **Groq Llama-70b Output:**

def deadband_filter(signal: list[float], threshold: float, apply_scaling: bool = False) -> list[float]:
    """
    Implements deadband filtering with optional scaling.

    Parameters:
    signal (list[float]): input signal
    threshold (float): deadband threshold, must be >= 0
    apply_scaling (bool): if True, apply scaling outside deadband

    Returns:
    list[float]: filtered signal
    """
    if threshold < 0:
        raise ValueError("threshold must be non-negative")

    output = []
    for x in signal:
        abs_x = abs(x)
        if abs_x <= threshold:
            output.append(0.0)
        else:
            if apply_scaling:
                # P2 scaling: subtract threshold magnitude
                scaled = (abs_x - threshold) * (1.0 if x >= 0 else -1.0)
                output.append(scaled)
            else:
                output.append(x)
    return output
```

**Comparison:**
- **Correctness:** Both implement the spec correctly. Both include threshold validation.
- **Conciseness:** Nearly identical. DeepSeek uses `sign` variable; Groq uses inline ternary.
- **Docstring:** DeepSeek's docstring explicitly mentions P0/P1/P2, aligning with fleet doctrine. Groq's is more generic.
- **Edge Cases:** Both handle negative samples correctly.

## **3. Function 2: `parse_fleet_log`**

**Prompt to both models:**
```
Write a Python function called `parse_fleet_log` that parses a structured log line from the Cocapn fleet.

Log format example: "[2026-04-20T08:15:32Z] [INFO] [oracle1] tile_accepted cycle=257"

Function signature:
def parse_fleet_log(line: str) -> dict[str, str]:

Rules:
- Extract timestamp (inside first brackets).
- Extract log level (inside second brackets): DEBUG, INFO, WARN, ERROR.
- Extract service name (inside third brackets): e.g., oracle1, jetsonclaw1, forgemaster.
- Extract message (everything after third bracket and space).
- If line doesn't match pattern, return dict with all keys set to "ERROR".

Return dictionary with keys: timestamp, level, service, message.

Use regex for robustness. Include a docstring.
```

### **DeepSeek-chat Output:**
```python
import re

def parse_fleet_log(line: str) -> dict[str, str]:
    """
    Parse a structured fleet log line.

    Format: [timestamp] [level] [service] message

    Args:
        line: Log line string.

    Returns:
        Dictionary with keys: timestamp, level, service, message.
        If format invalid, all values are "ERROR".
    """
    pattern = r'^\[([^\]]+)\]\s+\[([^\]]+)\]\s+\[([^\]]+)\]\s+(.+)$'
    match = re.match(pattern, line)
    if match:
        return {
            "timestamp": match.group(1),
            "level": match.group(2),
            "service": match.group(3),
            "message": match.group(4)
        }
    else:
        return {"timestamp": "ERROR", "level": "ERROR", "service": "ERROR", "message": "ERROR"}
```

### **Groq Llama-70b Output:**
```python
import re

def parse_fleet_log(line: str) -> dict[str, str]:
    """
    Parses a fleet log line into its components.

    Expected format: [timestamp] [level] [service] message

    Args:
        line: A string log line.

    Returns:
        A dict with keys: timestamp, level, service, message.
        If parsing fails, all values are "ERROR".
    """
    # Regex matches three bracketed groups then remaining message
    regex = r'\[([^\]]+)\]\s+\[([^\]]+)\]\s+\[([^\]]+)\]\s+(.+)'
    m = re.match(regex, line)
    if m:
        return {
            "timestamp": m.group(1),
            "level": m.group(2),
            "service": m.group(3),
            "message": m.group(4)
        }
    else:
        return {k: "ERROR" for k in ("timestamp", "level", "service", "message")}
```

**Comparison:**
- **Correctness:** Both regex patterns are identical and correct. Both handle error case.
- **Conciseness:** Groq uses a dict comprehension for error case, which is slightly more elegant.
- **Docstring:** Both are clear.
- **Robustness:** Both use `re.match` (good for start-of-line). DeepSeek's pattern uses `^` and `$` anchors explicitly.
