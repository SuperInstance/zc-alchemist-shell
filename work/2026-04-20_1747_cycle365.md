## **2. Function 1: `deadband_filter`**

**Prompt to both models:**
```
Write a Python function `deadband_filter(values, deadband=0.1)` that implements the Deadband Protocol.

Input: `values` is a list of floats. `deadband` is a positive float.

P0 (negative space): If any value is NaN or infinite, raise ValueError.
P1 (safe channel): Filter out values where abs(value) < deadband (these are in the "dead zone").
P2 (optimize): Return the filtered list, preserving original order.

Example:
deadband_filter([0.0, 0.05, -0.2, 0.3], deadband=0.1) → [-0.2, 0.3]
```

### **DeepSeek-chat output:**
```python
import math

def deadband_filter(values, deadband=0.1):
    """
    Apply Deadband Protocol filtering to a list of float values.
    
    P0: Reject NaN or infinite values.
    P1: Filter out values inside the deadband (|value| < deadband).
    P2: Return filtered list in original order.
    """
    # P0: Check for invalid values
    for v in values:
        if not isinstance(v, (int, float)):
            raise ValueError(f"Value {v} is not a number")
        if math.isnan(v) or math.isinf(v):
            raise ValueError(f"Value {v} is NaN or infinite")
    
    # P1: Filter deadband zone
    filtered = [v for v in values if abs(v) >= deadband]
    
    return filtered
```

### **Groq Llama-70b output:**

```python
def deadband_filter(values, deadband=0.1):
    """
    Implements the Deadband Protocol.
    P0: Check for NaN/inf values.
    P1: Remove values within deadband.
    P2: Return filtered list.
    """
    import math
    
    # P0: Validate inputs
    if not isinstance(values, list):
        raise TypeError("values must be a list")
    if not isinstance(deadband, (int, float)) or deadband <= 0:
        raise ValueError("deadband must be a positive number")
    
    filtered = []
    for val in values:
        # Check if numeric
        if not isinstance(val, (int, float)):
            raise ValueError(f"Non-numeric value: {val}")
        # P0: Check NaN/inf
        if math.isnan(val) or math.isinf(val):
            raise ValueError(f"Invalid value (NaN/inf): {val}")
        # P1: Apply deadband
        if abs(val) >= deadband:
            filtered.append(val)
    return filtered
```

### **Comparison:**
- **Correctness:** Both handle NaN/inf and filtering correctly. Groq adds extra input validation (list type, deadband positive).
- **Conciseness:** DeepSeek uses list comprehension; Groq uses explicit loop. DeepSeek slightly more concise.
- **Readability:** Both clear. DeepSeek docstring references P0/P1/P2 explicitly.
- **Edge cases:** Groq validates `deadband` type/positivity; DeepSeek does not. Both raise `ValueError` on non‑numeric elements.

**Verdict:** DeepSeek’s version is more direct; Groq’s is more defensive. For fleet use, Groq’s extra checks may be beneficial (P0 thinking).

## **3. Function 2: `parse_fleet_log`**

**Prompt:**
```
Write a Python function `parse_fleet_log(line)` that parses a fleet log line.

Log format: "2026-04-20 17:46:23 [AGENT] [CYCLE] message"
Where:
- timestamp: YYYY-MM-DD HH:MM:SS
- AGENT: uppercase word (e.g., "ALCHEMIST")
- CYCLE: integer
- message: rest of line

Return a dict with keys: timestamp (as datetime object), agent (str), cycle (int), message (str).
If line doesn't match format, return None.
Use datetime.strptime for parsing.
```

### **DeepSeek-chat output:**
```python
from datetime import datetime
import re

def parse_fleet_log(line):
    """
    Parse a fleet log line into structured dict.
    Format: "YYYY-MM-DD HH:MM:SS [AGENT] [CYCLE] message"
    """
    pattern = r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) \[([A-Z]+)\] \[(\d+)\] (.+)$'
    match = re.match(pattern, line.strip())
    if not match:
        return None
    
    timestamp_str, agent, cycle_str, message = match.groups()
    
    try:
        timestamp = datetime.strptime(timestamp_str, "%Y-%m-%d %H:%M:%S")
        cycle = int(cycle_str)
    except ValueError:
        return None
    
    return {
        "timestamp": timestamp,
        "agent": agent,
        "cycle": cycle,
        "message": message.strip()
    }
```

### **Groq Llama-70b output:**

```python
import re
from datetime import datetime

def parse_fleet_log(line):
    """
    Parses a standardized fleet log line.
    Expected format: "2026-04-20 17:46:23 [AGENT] [CYCLE] message"
    Returns dict with timestamp (datetime), agent (str), cycle (int), message (str).
    Returns None if format invalid.
    """
    if not line or not isinstance(line, str):
        return None
    
    # Regex to capture components
    log_regex = re.compile(
        r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) \[([A-Z]+)\] \[(\d+)\] (.+)$'
    )
    
    m = log_regex.match(line.strip())
    if not m:
        return None
    
    date_str, agent, cycle_str, msg = m.groups()
    
    try:
        ts = datetime.strptime(date_str, "%Y-%m-%d %H:%M:%S")
        cyc = int(cycle_str)
    except Exception:
        return None
    
    return {
        "timestamp": ts,
        "agent": agent,
        "cycle": cyc,
        "message": msg.strip()
    }
```

### **Comparison:**
- **Correctness:** Both implement regex identically. Both handle parsing errors.
- **Conciseness:** Nearly identical. Groq adds explicit `if not line` check.
- **Readability:** Both clear. DeepSeek uses `match.groups()` directly; Groq uses `m.groups()`.
- **Edge cases:** Groq checks `line` type and emptiness; DeepSeek relies on regex match failing.

**Verdict:** Tie. Both produce robust parsers. Groq slightly more defensive.
