### **Function 1: `clean_and_normalize_text`**

**Prompt:**
"Write a Python function `clean_and_normalize_text(text: str) -> str`. It should: 1) collapse multiple whitespace to a single space, 2) convert to lowercase, 3) remove all non-alphanumeric characters except basic punctuation .!?, and 4) expand common English contractions (e.g., "don't" -> "do not"). Handle empty string input. Include type hints and a brief docstring."

#### **DeepSeek-chat Output:**
```python
import re

def clean_and_normalize_text(text: str) -> str:
    """
    Clean and normalize input text by:
    1. Collapsing multiple whitespace to single space.
    2. Converting to lowercase.
    3. Removing non-alphanumeric chars except .!?
    4. Expanding common contractions.
    
    Args:
        text (str): Input text, can be empty.
    
    Returns:
        str: Cleaned and normalized text.
    """
    if not text:
        return ""
    
    # Step 1: Collapse whitespace
    text = re.sub(r'\s+', ' ', text)
    
    # Step 2: Lowercase
    text = text.lower()
    
    # Step 3: Remove unwanted chars (keep alphanumeric and .!?)
    text = re.sub(r'[^a-z0-9.!? ]', '', text)
    
    # Step 4: Expand contractions
    contractions = {
        "don't": "do not",
        "can't": "cannot",
        "won't": "will not",
        "it's": "it is",
        "i'm": "i am",
        "you're": "you are",
        "they're": "they are",
        "we're": "we are",
        "isn't": "is not",
        "aren't": "are not",
        "wasn't": "was not",
        "weren't": "were not",
        "haven't": "have not",
        "hasn't": "has not",
        "hadn't": "had not",
        "wouldn't": "would not",
        "couldn't": "could not",
        "shouldn't": "should not"
    }
    
    for cont, expanded in contractions.items():
        text = text.replace(cont, expanded)
    
    # Final whitespace strip
    return text.strip()
```

    - expanding contractions like don't -> do not
    """

        "they're": "they are",
        "we're": "we are",

*   **Correctness:** Both are functionally correct for the core task. DeepSeek's contraction list is more comprehensive.
*   **Robustness:** DeepSeek checks `if not text:` which handles `None` and empty string. Groq checks `if text is None or len(text) == 0:`, which is more explicit but slightly redundant (`len("")==0`). Both are adequate.
*   **Clarity:** DeepSeek's docstring is more detailed, explicitly listing steps. Groq's is concise. Both are readable.
*   **Idiomatic Style:** Both use type hints. DeepSeek's final `strip()` is good practice. Groq's initial `strip().lower()` before whitespace collapse could theoretically be less efficient if there are leading/trailing spaces, but it's negligible.
*   **Edge:** DeepSeek provides a slightly more thorough solution.
