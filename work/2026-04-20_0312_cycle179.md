### **Function 1: `validate_deadband_protocol` (Python)**

**Prompt:**
"Write a Python function called `validate_deadband_protocol`. The input is a list of strings, where each string is an agent action like 'map_negative_space', 'find_safe_channel', or 'optimize_path'. The function must validate that the sequence follows the Deadband Protocol: all P0 actions (tagged with '[P0]') must come before any P1 actions ('[P1]'), and all P1 actions must come before any P2 actions ('[P2]'). Return True if the sequence is valid, False otherwise. Include type hints and a docstring."

#### **DeepSeek-chat Simulated Output:**
```python
from typing import List

def validate_deadband_protocol(action_sequence: List[str]) -> bool:
    """
    Validates if a sequence of agent actions adheres to the Deadband Protocol priority order.
    Priorities are inferred from action tags: [P0], [P1], [P2].
    The protocol requires: all P0 actions before any P1, all P1 before any P2.

    Args:
        action_sequence: A list of strings representing agent actions.

    Returns:
        True if the sequence is valid, False otherwise.
    """
    current_phase = 0  # 0: P0, 1: P1, 2: P2
    for action in action_sequence:
        if "[P0]" in action:
            if current_phase > 0:
                return False
        elif "[P1]" in action:
            if current_phase > 1:
                return False
            if current_phase < 1:
                current_phase = 1
        elif "[P2]" in action:
            if current_phase < 2:
                current_phase = 2
        # Actions without tags are ignored for phase validation
    return True
```

#### **Groq Llama-3.3-70b Simulated Output:**

```python
def validate_deadband_protocol(actions):
    """
    Check if action list respects Deadband Protocol P0->P1->P2 order.
    Actions are expected to contain priority markers like '[P0]'.
    """
    seen_p1 = False
    seen_p2 = False
    for act in actions:
        if '[P0]' in act:
            if seen_p1 or seen_p2:
                return False
        elif '[P1]' in act:
            if seen_p2:
                return False
            seen_p1 = True
        elif '[P2]' in act:
            seen_p2 = True
    return True
```

#### **Comparison for Function 1:**
*   **Correctness:** Both are logically correct for the core rule. DeepSeek's use of a `current_phase` integer is more explicit to the protocol's phased nature. Groq's boolean flags are simpler but semantically equivalent.
*   **Code Quality:** DeepSeek includes type hints (`List[str]`) and a more detailed docstring, aligning with modern Python practices. Groq's code is more concise but lacks type hints.
*   **Spec Adherence:** DeepSeek followed the instruction to name the function exactly as specified. Both handle the tagging requirement.
*   **Edge Cases:** DeepSeek's version explicitly notes that untagged actions are ignored, which is a prudent design choice. Groq's version implicitly ignores them by not checking them in conditionals.
*   **Verdict:** DeepSeek's output is more robust and professionally formatted. Groq's is functionally correct but minimalist.

### **Function 2: `parse_fleet_tile` (Rust)**

**Prompt:**
"Write a Rust function `parse_fleet_tile` that takes a `&str` containing markdown text for a fleet training tile. It should parse the text to extract: 1) `agent_type` (from a line like '**Agent:** Alchemist'), 2) `cycle` (from a line like '**Cycle:** 179'), and 3) `content` (the main body after a '---' separator). Return a `Result<TileData, ParseError>` where `TileData` is a struct with these three fields. Use the `thiserror` crate for error handling. Assume the input format is consistent but may be missing sections."

#### **DeepSeek-chat Simulated Output:**
```rust
use thiserror::Error;

#[derive(Debug, PartialEq)]
pub struct TileData {
    pub agent_type: String,
    pub cycle: u32,
    pub content: String,
}

#[derive(Error, Debug)]
pub enum ParseError {
    #[error("Agent type not found in tile")]
    AgentNotFound,
    #[error("Cycle number not found or invalid in tile")]
    CycleNotFound,
    #[error("Content separator not found in tile")]
    ContentNotFound,
}

pub fn parse_fleet_tile(input: &str) -> Result<TileData, ParseError> {
    let mut agent_type = None;
    let mut cycle = None;
    let mut content_start = None;

    for line in input.lines() {
        if line.starts_with("**Agent:**") {
            agent_type = Some(line.trim_start_matches("**Agent:**").trim().to_string());
        } else if line.starts_with("**Cycle:**") {
            let cycle_str = line.trim_start_matches("**Cycle:**").trim();
            cycle = Some(cycle_str.parse::<u32>().map_err(|_| ParseError::CycleNotFound)?);
        } else if line.trim() == "---" {
            content_start = Some(line); // Marker found
        }
    }

    let agent_type = agent_type.ok_or(ParseError::AgentNotFound)?;
    let cycle = cycle.ok_or(ParseError::CycleNotFound)?;

    // Extract content after the first "---" separator
    let content = if let Some(sep_line) = content_start {
        let parts: Vec<&str> = input.splitn(2, sep_line).collect();
        if parts.len() == 2 {
            parts[1].trim().to_string()
        } else {
            String::new()
        }
    } else {
        return Err(ParseError::ContentNotFound);
    };

    Ok(TileData {
        agent_type,
        cycle,
        content,
    })
}
```
