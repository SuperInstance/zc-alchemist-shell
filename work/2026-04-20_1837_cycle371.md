## **2. Function 1: `parse_log_line`**

**Prompt to both models:**
```
Write a Python function `parse_log_line` that takes a string like "2026-04-20 18:34:23 INFO [module] message" and returns a dict with keys: timestamp (datetime), level (str), module (str), message (str). Handle malformed input gracefully. Include a docstring and example.
```

### **DeepSeek-chat output:**
```python
import re
from datetime import datetime

def parse_log_line(line: str) -> dict:
    """
    Parse a structured log line into its components.
    
    Expected format: "YYYY-MM-DD HH:MM:SS LEVEL [MODULE] MESSAGE"
    
    Args:
        line (str): Log line string.
    
    Returns:
        dict: Parsed components with keys: timestamp (datetime), level (str),
              module (str), message (str). Returns None if parsing fails.
    
    Example:
        >>> parse_log_line("2026-04-20 18:34:23 INFO [network] Connection established")
        {'timestamp': datetime.datetime(2026, 4, 20, 18, 34, 23),
         'level': 'INFO',
         'module': 'network',
         'message': 'Connection established'}
    """
    pattern = r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) (\w+) \[([^\]]+)\] (.+)$'
    match = re.match(pattern, line.strip())
    if not match:
        return None
    
    timestamp_str, level, module, message = match.groups()
    try:
        timestamp = datetime.strptime(timestamp_str, '%Y-%m-%d %H:%M:%S')
    except ValueError:
        return None
    
    return {
        'timestamp': timestamp,
        'level': level,
        'module': module,
        'message': message.strip()
    }
```

### **Groq Llama-70b output:**

```python
import datetime
import re

def parse_log_line(log_line):
    """
    Parses a log line of format "2026-04-20 18:34:23 INFO [module] message"
    into a dictionary with keys: timestamp, level, module, message.
    
    Args:
        log_line (str): A string containing the log entry.
    
    Returns:
        dict or None: A dictionary with parsed components if successful,
                      None if the line is malformed.
    """
    # Regex pattern to match the log line format
    pattern = r'^(\d{4}-\d{2}-\d{2}) (\d{2}:\d{2}:\d{2}) (\w+) \[([^\]]+)\] (.+)$'
    match = re.match(pattern, log_line.strip())
    
    if not match:
        return None
    
    date_str, time_str, level, module, message = match.groups()
    
    try:
        # Combine date and time into a datetime object
        timestamp = datetime.datetime.strptime(f"{date_str} {time_str}", "%Y-%m-%d %H:%M:%S")
    except ValueError:
        return None
    
    return {
        "timestamp": timestamp,
        "level": level,
        "module": module,
        "message": message.strip()
    }
```

### **Comparison:**
- **Correctness**: Both handle the format. DeepSeek uses a single regex group for timestamp; Groq splits date/time. Both return `None` on failure.
- **Readability**: DeepSeek's docstring includes an example return value; Groq's is simpler. DeepSeek uses type hints.
- **Robustness**: DeepSeek's pattern expects a space between date and time in one group; Groq captures separately. Both equivalent.
- **Edge**: DeepSeek's `strip()` on message only; Groq strips whole line first. Minor difference.
- **Fleet alignment**: Neither uses fleet-specific patterns (not needed here).

**Verdict**: Tie. Both produce correct, readable code.

## **3. Function 2: `deadband_filter`**

**Prompt:**
```
Write a Python function `deadband_filter` that implements the Cocapn fleet's Deadband Protocol. Input: a list of action candidate dicts, each with 'name', 'p0_risk' (bool), 'p1_channel' (bool), 'p2_score' (float). Protocol: P0: reject if p0_risk True. P1: keep only if p1_channel True. P2: sort by p2_score descending. Return filtered and sorted list. Include docstring and example.
```

### **DeepSeek-chat output:**
```python
from typing import List, Dict, Any

def deadband_filter(candidates: List[Dict[str, Any]]) -> List[Dict[str, Any]]:
    """
    Apply the Deadband Protocol (P0→P1→P2) to filter and sort action candidates.
    
    P0: Eliminate candidates with p0_risk == True (negative space).
    P1: Keep only candidates with p1_channel == True (safe channels).
    P2: Sort remaining candidates by p2_score descending (optimization).
    
    Args:
        candidates: List of dicts with keys: 'name', 'p0_risk', 'p1_channel', 'p2_score'.
    
    Returns:
        Filtered and sorted list of candidates.
    
    Example:
        >>> candidates = [
        ...     {'name': 'A', 'p0_risk': False, 'p1_channel': True, 'p2_score': 0.5},
        ...     {'name': 'B', 'p0_risk': True, 'p1_channel': True, 'p2_score': 0.9},
        ...     {'name': 'C', 'p0_risk': False, 'p1_channel': False, 'p2_score': 0.8},
        ... ]
        >>> deadband_filter(candidates)
        [{'name': 'A', 'p0_risk': False, 'p1_channel': True, 'p2_score': 0.5}]
    """
    # P0: Map negative space
    p0_safe = [c for c in candidates if not c.get('p0_risk', False)]
    
    # P1: Find safe channels
    p1_valid = [c for c in p0_safe if c.get('p1_channel', False)]
    
    # P2: Optimize within channels
    p2_sorted = sorted(p1_valid, key=lambda x: x.get('p2_score', 0.0), reverse=True)
    
    return p2_sorted
```
