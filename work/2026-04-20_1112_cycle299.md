## **2. Function 1: deadband_filter()**

**Purpose:** Apply Deadband Protocol to a list of float sensor readings. P0: remove values that are NaN or outside physical range (-100.0 to 100.0). P1: filter values within deadband threshold (±0.5) as "stable". P2: return list of filtered values and stable count.

### **DeepSeek-chat output:**
```python
from typing import Tuple, List

def deadband_filter(readings: List[float], deadband: float = 0.5) -> Tuple[List[float], int]:
    """
    Apply Deadband Protocol to sensor readings.
    
    P0: Remove invalid readings (NaN, outside -100.0 to 100.0).
    P1: Identify readings within deadband threshold as stable.
    P2: Return filtered valid readings and count of stable readings.
    
    Args:
        readings: List of raw sensor readings.
        deadband: Absolute threshold for stability detection.
    
    Returns:
        Tuple of (filtered_readings, stable_count).
    """
    if not isinstance(readings, list):
        raise TypeError("readings must be a list")
    
    filtered = []
    stable_count = 0
    
    for val in readings:
        # P0: Check for invalid values
        if not isinstance(val, (int, float)):
            continue  # skip non-numeric
        if val != val:  # NaN check
            continue
        if val < -100.0 or val > 100.0:
            continue
        
        filtered.append(val)
        
        # P1: Check if within deadband
        if abs(val) <= deadband:
            stable_count += 1
    
    return filtered, stable_count
```

### **Groq Llama-70b output:**

```python
from typing import List, Tuple

def deadband_filter(readings: List[float], deadband: float = 0.5) -> Tuple[List[float], int]:
    """
    Implements the Deadband Protocol for sensor data filtering.
    
    P0: Remove values that are NaN or outside the operational range [-100.0, 100.0].
    P1: Count values that fall within the deadband interval [-deadband, deadband] as stable.
    P2: Return the cleaned list and the number of stable readings.
    
    Parameters:
        readings (List[float]): Input sensor readings.
        deadband (float): Deadband threshold, default 0.5.
    
    Returns:
        Tuple[List[float], int]: (filtered readings, stable count)
    """
    filtered_readings: List[float] = []
    stable: int = 0
    
    for reading in readings:
        # P0: Validate reading
        try:
            # Check if it's a number
            if not isinstance(reading, (int, float)):
                continue
            # Check for NaN
            if reading != reading:
                continue
            # Check range
            if reading < -100.0 or reading > 100.0:
                continue
        except Exception:
            continue
        
        filtered_readings.append(reading)
        
        # P1: Check stability
        if -deadband <= reading <= deadband:
            stable += 1
    
    return filtered_readings, stable
```

**Comparison:**
- **Correctness:** Both handle NaN, type checks, and range validation. DeepSeek uses `val != val` for NaN; Groq uses same pattern. Both implement deadband correctly.
- **Completeness:** Both have docstrings with P0/P1/P2. DeepSeek adds explicit `TypeError` for non-list input. Groq wraps validation in try-except (possibly overkill).
- **Fleet style:** DeepSeek's docstring explicitly labels P0/P1/P2 steps, matching fleet documentation style. Groq's is more generic.
- **Conciseness:** DeepSeek: 28 lines, Groq: 33 lines. DeepSeek slightly more compact.

## **3. Function 2: parse_fleet_log()**

**Purpose:** Parse a Cocapn fleet log line format: `[TIMESTAMP] [AGENT] [LEVEL] message`. Return dict with keys: timestamp, agent, level, message. Handle malformed lines gracefully.

### **DeepSeek-chat output:**
```python
import re
from typing import Optional, Dict

def parse_fleet_log(log_line: str) -> Optional[Dict[str, str]]:
    """
    Parse a standardized Cocapn fleet log line.
    
    Expected format: [TIMESTAMP] [AGENT] [LEVEL] message
    Example: [2026-04-20T11:30:00Z] [Alchemist] [INFO] Task completed.
    
    Args:
        log_line: A single log line string.
    
    Returns:
        Dictionary with keys: timestamp, agent, level, message.
        Returns None if line cannot be parsed.
    """
    if not isinstance(log_line, str):
        return None
    
    # Regex matches: [timestamp] [agent] [level] message
    pattern = r'^\[([^\]]+)\]\s+\[([^\]]+)\]\s+\[([^\]]+)\]\s+(.+)$'
    match = re.match(pattern, log_line.strip())
    
    if not match:
        return None
    
    timestamp, agent, level, message = match.groups()
    
    # Basic validation
    if not timestamp or not agent or not level or not message:
        return None
    
    return {
        "timestamp": timestamp.strip(),
        "agent": agent.strip(),
        "level": level.strip(),
        "message": message.strip()
    }
```

### **Groq Llama-70b output:**
```python
import re
from typing import Dict, Optional

def parse_fleet_log(log_line: str) -> Optional[Dict[str, str]]:
    """
    Parses a Cocapn fleet log line of the format:
    [TIMESTAMP] [AGENT] [LEVEL] message
    
    Args:
        log_line: The log line string to parse.
    
    Returns:
        A dictionary with keys 'timestamp', 'agent', 'level', 'message' 
        if parsing succeeds, else None.
    """
    if not log_line or not isinstance(log_line, str):
        return None
    
    stripped = log_line.strip()
    
    # Regular expression to extract the four components
    regex = r'^\[([^\]]+)\]\s+\[([^\]]+)\]\s+\[([^\]]+)\]\s+(.*)$'
    match = re.match(regex, stripped)
    
    if match is None:
        return None
    
    timestamp, agent, level, message = match.groups()
    
    # Ensure no component is empty
    if not (timestamp and agent and level):
        return None
    
    result = {
        "timestamp": timestamp,
        "agent": agent,
        "level": level,
        "message": message
    }
    
    return result
```

**Comparison:**
- **Correctness:** Both use identical regex pattern. Both handle None/empty input. DeepSeek also checks `not message` in validation; Groq only checks first three fields.
- **Completeness:** Both have docstrings with example. DeepSeek includes explicit example format. Groq's docstring is slightly less detailed.
- **Fleet style:** Both are defensive. DeepSeek adds `.strip()` on each field, which may be more robust against extra spaces.
- **Conciseness:** DeepSeek: 30 lines, Groq: 29 lines. Similar length.
