## **2. Function 1: `validate_deadband_protocol`**

**Prompt:**
```
Write a Python function `validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]`.
It validates if a list of protocol steps follows the Deadband Protocol's strict priority: P0 (map negative space) must come before any P1 (find safe channels), and P1 must come before any P2 (optimize). The list can contain only "P0", "P1", or "P2". Return (True, "") if valid, otherwise (False, "error message").

Include a docstring with examples:
- validate_deadband_protocol(["P0", "P1", "P2"]) -> (True, "")
- validate_deadband_protocol(["P1", "P0"]) -> (False, "P1 found before P0")
- validate_deadband_protocol(["P0", "P2"]) -> (False, "P2 found without preceding P1")
- validate_deadband_protocol(["P0", "P1", "P0"]) -> (False, "Priority regression: P0 after P1")
```

**DeepSeek-chat Output:**
```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """
    Validate if steps follow Deadband Protocol strict priority: P0 → P1 → P2.
    
    Args:
        steps: List of step strings, each must be "P0", "P1", or "P2".
    
    Returns:
        Tuple of (is_valid: bool, error_message: str).
    
    Examples:
        >>> validate_deadband_protocol(["P0", "P1", "P2"])
        (True, "")
        >>> validate_deadband_protocol(["P1", "P0"])
        (False, "P1 found before P0")
        >>> validate_deadband_protocol(["P0", "P2"])
        (False, "P2 found without preceding P1")
        >>> validate_deadband_protocol(["P0", "P1", "P0"])
        (False, "Priority regression: P0 after P1")
    """
    valid_steps = {"P0", "P1", "P2"}
    highest_seen = -1  # 0=P0, 1=P1, 2=P2
    step_order = {"P0": 0, "P1": 1, "P2": 2}
    
    for i, step in enumerate(steps):
        if step not in valid_steps:
            return False, f"Invalid step '{step}' at position {i}"
        
        current_order = step_order[step]
        
        # Check for regression
        if current_order < highest_seen:
            return False, f"Priority regression: {step} after {list(step_order.keys())[highest_seen]}"
        
        # Check for missing prerequisite
        if current_order > 0 and highest_seen < 0:
            return False, f"P{current_order} found without preceding P0"
        if current_order > 1 and highest_seen < 1:
            return False, f"P2 found without preceding P1"
        
        highest_seen = current_order
    
    return True, ""
```

**Groq Llama-70b Output (Simulated):**

def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """
    Validates Deadband Protocol sequence.
    
    Rules:
    1. Only P0, P1, P2 allowed.
    2. P0 must come before P1.
    3. P1 must come before P2.
    4. No going back to earlier priority after later one.
    
    Examples:
        >>> validate_deadband_protocol(["P0", "P1", "P2"])
        (True, "")
        >>> validate_deadband_protocol(["P1", "P0"])
        (False, "P1 appears before P0")
        >>> validate_deadband_protocol(["P0", "P2"])
        (False, "P2 appears without P1")
        >>> validate_deadband_protocol(["P0", "P1", "P0"])
        (False, "Priority reversal: P0 after P1")
    """
    allowed = {"P0", "P1", "P2"}
    seen_p0 = False
    seen_p1 = False
    
    for idx, step in enumerate(steps):
        if step not in allowed:
            return False, f"Invalid step '{step}' at index {idx}"
        
        if step == "P0":
            seen_p0 = True
            if seen_p1:
                return False, "Priority reversal: P0 after P1"
        elif step == "P1":
            if not seen_p0:
                return False, "P1 appears before P0"
            seen_p1 = True
        elif step == "P2":
            if not seen_p1:
                return False, "P2 appears without P1"
            if not seen_p0:
                return False, "P2 appears without P0"  # Redundant but safe
    
    return True, ""
```

**Comparison:**
*   **Correctness:** Both handle the core rules. DeepSeek uses a numeric mapping (`step_order`) and tracks `highest_seen`, which elegantly catches regressions and missing prerequisites in one check. Groq uses boolean flags (`seen_p0`, `seen_p1`), which is also correct but has a redundant check for P2 without P0.
*   **Style:** DeepSeek's docstring is more detailed with a clear `Args`/`Returns` section. Groq's is a bulleted list of rules.
*   **Edge Cases:** Both reject invalid steps. DeepSeek's regression message is more specific (`"Priority regression: P0 after P1"`). Groq's `"Priority reversal"` is similar.
*   **Adherence to Spec:** Both return the exact requested tuple format.

**Verdict:** DeepSeek's solution is slightly more elegant and extensible (adding P3 would be trivial). Groq's is perfectly functional and readable. **DeepSeek wins on algorithmic elegance.**
