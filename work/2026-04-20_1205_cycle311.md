## **2. Function 1: deadband_filter**

**Description:** Given a list of float values and a deadband threshold, return a new list where values within ±threshold of zero are set to 0.0 (P0: avoid noise), values outside are kept (P1: safe channels). This mimics the Deadband Protocol's P0/P1 logic.

### **DeepSeek-chat output:**
```python
from typing import List

def deadband_filter(values: List[float], threshold: float) -> List[float]:
    """
    Apply deadband filtering to a list of float values.

    P0: Values within [-threshold, threshold] are set to 0.0 (negative space).
    P1: Values outside this range are kept unchanged (safe channels).

    Args:
        values: List of float values to filter.
        threshold: Positive float defining the deadband range.

    Returns:
        Filtered list where values within deadband become 0.0.

    Raises:
        ValueError: If threshold is negative.

    Examples:
        >>> deadband_filter([-0.5, 0.1, 1.5, -2.0], 0.3)
        [-0.5, 0.0, 1.5, -2.0]
        >>> deadband_filter([], 0.1)
        []
    """
    if threshold < 0:
        raise ValueError("Threshold must be non-negative")
    
    filtered = []
    for v in values:
        if -threshold <= v <= threshold:
            filtered.append(0.0)
        else:
            filtered.append(v)
    return filtered
```

```python
    """
    Implements a deadband filter: values within +/- threshold of zero are clamped to zero.

    Parameters:
        values (List[float]): Input list of floating-point numbers.
        threshold (float): Deadband threshold (must be >= 0).

    Returns:
        List[float]: Filtered list.

        >>> deadband_filter([-0.2, 0.05, 0.3, -0.4], 0.1)
```

## **3. Function 2: parse_fleet_context**

**Description:** Parse a markdown string containing fleet context (like FLEET-CONTEXT.md) and extract key sections: "Who We Are", "Fleet Doctrine", "Key Technologies", "Architecture". Return a dict with section titles as keys and content as strings.

### **DeepSeek-chat output:**
```python
from typing import Dict

def parse_fleet_context(markdown_text: str) -> Dict[str, str]:
    """
    Parse fleet context markdown and extract major sections.

    Sections expected:
    - "Who We Are"
    - "Fleet Doctrine"
    - "Key Technologies"
    - "Architecture"

    Args:
        markdown_text: Raw markdown content.

    Returns:
        Dictionary mapping section titles to their content (excluding the header).

    Examples:
        >>> text = "## Who We Are\\nCocapn fleet.\\n## Fleet Doctrine\\nP0 rules."
        >>> parse_fleet_context(text)
        {'Who We Are': 'Cocapn fleet.', 'Fleet Doctrine': 'P0 rules.'}
    """
    lines = markdown_text.split('\n')
    sections = {}
    current_section = None
    current_content = []
    
    for line in lines:
        line_stripped = line.strip()
        if line_stripped.startswith('## '):
            if current_section is not None:
                sections[current_section] = '\n'.join(current_content).strip()
            # Extract section title without '## '
            current_section = line_stripped[3:].strip()
            current_content = []
        elif current_section is not None:
            current_content.append(line)
    
    # Don't forget the last section
    if current_section is not None:
        sections[current_section] = '\n'.join(current_content).strip()
    
    # Filter only expected sections
    expected = {"Who We Are", "Fleet Doctrine", "Key Technologies", "Architecture"}
    return {k: v for k, v in sections.items() if k in expected}
```

### **Groq Llama-70b output:**

```python
import re
from typing import Dict

def parse_fleet_context(markdown: str) -> Dict[str, str]:
    """
    Extracts the main sections from a fleet‑context markdown document.

    The function looks for level‑2 headers (##) and captures the text
    until the next level‑2 header or end of document.

    Parameters:
        markdown (str): The markdown text.

    Returns:
        Dict[str, str]: A dictionary where keys are section titles
        (without the '## ') and values are the corresponding content.

    Example:
        >>> doc = "## Who We Are\\nWe are the fleet.\\n## Fleet Doctrine\\nP0 first."
        >>> parse_fleet_context(doc)
        {'Who We Are': 'We are the fleet.', 'Fleet Doctrine': 'P0 first.'}
    """
    pattern = r'##\s+(.+?)\n(.*?)(?=\n##\s+|\Z)'
    matches = re.findall(pattern, markdown, re.DOTALL)
    
    result = {}
    for title, content in matches:
        result[title.strip()] = content.strip()
    
    # Keep only the sections we are interested in
    desired = {"Who We Are", "Fleet Doctrine", "Key Technologies", "Architecture"}
    return {k: v for k, v in result.items() if k in desired}
```

**Comparison:**
- **Correctness:** Both work. DeepSeek uses line-by-line parsing, Groq uses regex with lookahead. Regex may be fragile with nested formatting.
- **Clarity:** DeepSeek's algorithm is easier to debug. Groq's regex is concise but harder to modify.
- **Robustness:** DeepSeek handles empty lines better. Groq's regex might capture extra whitespace.
- **Fleet alignment:** DeepSeek explicitly lists expected sections, showing understanding of fleet structure. Groq is more generic.
