## **3. Function 1: deadband_filter()**

**Problem Description:**  
Implement `deadband_filter(readings: List[float], deadband: float) -> List[float]`.  
Given a list of sensor readings, apply a deadband filter: if a reading changes less than `deadband` from the previous *output* value, keep the previous output; otherwise, pass the new reading. The first reading always passes. This implements hysteresis.

### **DeepSeek-chat Output:**
```python
from typing import List

def deadband_filter(readings: List[float], deadband: float) -> List[float]:
    """
    Apply a deadband filter with hysteresis to a list of sensor readings.
    
    The filter only outputs a new value if it differs from the last output
    by at least `deadband`. Otherwise, the last output is repeated.
    
    Args:
        readings: List of sensor readings.
        deadband: Minimum change required to update output.
    
    Returns:
        Filtered list of readings.
    
    Example:
        >>> deadband_filter([1.0, 1.2, 1.3, 2.0], 0.5)
        [1.0, 1.0, 1.0, 2.0]
    """
    if not readings:
        return []
    
    filtered = [readings[0]]
    last_output = readings[0]
    
    for reading in readings[1:]:
        if abs(reading - last_output) >= deadband:
            filtered.append(reading)
            last_output = reading
        else:
            filtered.append(last_output)
    
    return filtered
```

### **Groq Llama-70b Output:**

```python
    """
    Apply a deadband filter to a list of readings.
    
    Parameters:
    readings (List[float]): Input list of sensor readings.
    deadband (float): Deadband threshold.
    
    Returns:
    List[float]: Filtered readings.
    
    Example:
    >>> deadband_filter([1.0, 1.1, 1.2, 5.0], 1.0)
    [1.0, 1.0, 1.0, 5.0]
    """

            result.append(val)
            prev = val

    return result
```

- **Correctness:** Both implement the hysteresis logic correctly.
- **Style:** DeepSeek includes a more detailed docstring with explicit `Args`/`Returns` and a clearer example. Groq uses `Parameters`/`Returns`.
- **Edge Cases:** Both handle empty list.
- **Variable Names:** DeepSeek uses `filtered`, `last_output`; Groq uses `result`, `prev`. Both are clear.
- **Adherence to Spec:** Both match signature and type hints.

## **4. Function 2: constraint_snap()**

**Problem Description:**  
Implement `constraint_snap(point: Tuple[float, float], constraints: List[Tuple[str, float]]) -> Tuple[float, float]`.  
Given a 2D point (x, y) and a list of constraints where each is (`axis`, `value`) with `axis` in `['x', 'y']`, snap the point to the nearest constraint per axis. If multiple constraints for same axis, choose closest. If no constraint for an axis, keep original coordinate.

### **DeepSeek-chat Output:**
```python
from typing import List, Tuple

def constraint_snap(point: Tuple[float, float], constraints: List[Tuple[str, float]]) -> Tuple[float, float]:
    """
    Snap a 2D point to the nearest constraint on each axis.
    
    Args:
        point: Original (x, y) coordinates.
        constraints: List of (axis, value) where axis is 'x' or 'y'.
    
    Returns:
        Snapped (x, y) coordinates.
    
    Example:
        >>> constraint_snap((3.0, 4.0), [('x', 1.0), ('x', 2.5), ('y', 5.0)])
        (2.5, 5.0)
    """
    x, y = point
    x_constraints = [val for axis, val in constraints if axis == 'x']
    y_constraints = [val for val for axis, val in constraints if axis == 'y']
    
    # Snap x
    if x_constraints:
        x = min(x_constraints, key=lambda c: abs(c - x))
    
    # Snap y
    if y_constraints:
        y = min(y_constraints, key=lambda c: abs(c - y))
    
    return (x, y)
```

### **Groq Llama-70b Output:**

```python
    """

    Parameters:
    point (Tuple[float, float]): (x, y)
    constraints (List[Tuple[str, float]]): List of (axis, value)
    
    Returns:

    >>> constraint_snap((3.0, 4.0), [('x', 1.0), ('x', 2.5), ('y', 5.0)])
    (2.5, 5.0)
    """

    y_vals = [val for ax, val in constraints if ax == 'y']

        # find closest x constraint
        best_x = min(x_vals, key=lambda v: abs(v - x))
        x = best_x
    if y_vals:
        best_y = min(y_vals, key=lambda v: abs(v - y))
        y = best_y
    
    return (x, y)
```

- **Clarity:** Groq uses `x_vals`/`y_vals`; DeepSeek (after correction) similar.
- **Docstring:** DeepSeek's docstring is more verbose; Groq's is concise.
- **Edge Cases:** Both handle empty constraint lists correctly.
- **Error Handling:** Neither validates axis strings beyond 'x'/'y'.
