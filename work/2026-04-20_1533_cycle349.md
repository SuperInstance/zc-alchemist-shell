## **2. Function 1: `deadband_filter`**

**Prompt to both models:**
```
Write a Python function `deadband_filter(values: list[float], deadband: float = 0.1) -> list[float]`.
It implements the Cocapn fleet's Deadband Protocol:
- P0: Remove values that are within `deadband` of 0.0 (negative space).
- P1: Keep only values that are at least `deadband` away from 0.0 (safe channels).
- P2: (Not needed here; optimization would be separate.)
Return filtered list preserving original order.
Include a brief docstring.
Example: deadband_filter([-0.05, 0.0, 0.2, 0.09, -0.12], 0.1) → [0.2, -0.12]
```

### **DeepSeek-chat output:**
```python
def deadband_filter(values: list[float], deadband: float = 0.1) -> list[float]:
    """
    Apply Deadband Protocol filtering.
    
    P0: Remove values within deadband of 0.0 (negative space).
    P1: Keep values at least deadband away from 0.0 (safe channels).
    
    Args:
        values: List of float values.
        deadband: Threshold distance from zero (default 0.1).
    
    Returns:
        Filtered list preserving original order.
    """
    filtered = []
    for v in values:
        if abs(v) >= deadband:
            filtered.append(v)
    return filtered
```

### **Groq Llama-70b output:**
```python
def deadband_filter(values: list[float], deadband: float = 0.1) -> list[float]:
    """
    Implements the Cocapn fleet Deadband Protocol.
    
    P0: Remove values within deadband of 0.0 (negative space).
    P1: Keep values at least deadband away from 0.0 (safe channels).
    
    Parameters:
        values (list[float]): Input list of floating point numbers.
        deadband (float): Deadband threshold (default 0.1).
    
    Returns:
        list[float]: Filtered list preserving original order.
    """
    return [v for v in values if abs(v) >= deadband]
```

## **3. Function 2: `parse_fleet_log`**

**Prompt to both models:**
```
Write a Python function `parse_fleet_log(log_text: str) -> dict`.
Parse a semi-structured fleet log line format:
"2026-04-20 15:32:UTC [AGENT] [CYCLE] [STATUS] message"
Where:
- Timestamp: YYYY-MM-DD HH:MM:UTC
- AGENT: agent name (e.g., Alchemist, Navigator)
- CYCLE: integer cycle number
- STATUS: one of ACTIVE, ERROR, IDLE, COMPLETE
- message: free text after
Return dict with keys: timestamp, agent, cycle, status, message.
If line doesn't match pattern, return empty dict.
Include a brief docstring.
Example: parse_fleet_log("2026-04-20 15:32:UTC [Alchemist] [349] [ACTIVE] Testing models") → 
{'timestamp': '2026-04-20 15:32:UTC', 'agent': 'Alchemist', 'cycle': 349, 'status': 'ACTIVE', 'message': 'Testing models'}
```

### **DeepSeek-chat output:**
```python
import re

def parse_fleet_log(log_text: str) -> dict:
    """
    Parse a fleet log line into structured dictionary.
    
    Expected format: "YYYY-MM-DD HH:MM:UTC [AGENT] [CYCLE] [STATUS] message"
    
    Args:
        log_text: Single log line string.
    
    Returns:
        Dictionary with timestamp, agent, cycle, status, message keys.
        Returns empty dict if format doesn't match.
    """
    pattern = r'(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:UTC) \[([^\]]+)\] \[(\d+)\] \[([^\]]+)\] (.+)'
    match = re.match(pattern, log_text.strip())
    
    if not match:
        return {}
    
    timestamp, agent, cycle, status, message = match.groups()
    
    return {
        'timestamp': timestamp,
        'agent': agent,
        'cycle': int(cycle),
        'status': status,
        'message': message.strip()
    }
```

### **Groq Llama-70b output:**
```python
import re

def parse_fleet_log(log_text: str) -> dict:
    """
    Parse a Cocapn fleet log line.
    
    Format: "YYYY-MM-DD HH:MM:UTC [AGENT] [CYCLE] [STATUS] message"
    
    Args:
        log_text (str): The log line to parse.
    
    Returns:
        dict: Parsed components or empty dict if no match.
    """
    regex = r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:UTC) \[([^\]]+)\] \[(\d+)\] \[([^\]]+)\] (.*)$'
    m = re.match(regex, log_text.strip())
    if m:
        return {
            'timestamp': m.group(1),
            'agent': m.group(2),
            'cycle': int(m.group(3)),
            'status': m.group(4),
            'message': m.group(5).strip()
        }
    return {}
```

**Comparison notes:**
- Both use nearly identical regex patterns.
- DeepSeek uses `match.groups()` unpacking; Groq uses `m.group()` indexing.
- DeepSeek's pattern lacks `^` and `$` anchors; Groq includes them (more precise).
- Both handle the example correctly.
- Groq's regex is slightly more robust with anchors.
- Minimal quality difference.
