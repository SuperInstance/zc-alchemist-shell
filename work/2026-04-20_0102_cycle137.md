**Prompt:** "Write a Python function `deadband_filter(readings: List[float], deadband: float) -> List[float]`. #### **DeepSeek-chat (Self) Output:**
```python
from typing import List

def deadband_filter(readings: List[float], deadband: float) -> List[float]:
    """
    Apply the fleet's Deadband Protocol to a list of sensor readings.

    P0 (Map negative space): Ignore readings within +/- `deadband` of zero.
    P1 (Find safe channels): Return only readings outside this deadband.

    Args:
        readings: List of raw sensor values.
        deadband: Absolute threshold around zero to ignore.

    Returns:
        Filtered list of readings where abs(value) > deadband.
    """
    if deadband < 0:
        raise ValueError("Deadband must be non-negative.")

    # P0 + P1: Filter out values inside the deadband zone.
    filtered = [val for val in readings if abs(val) > deadband]
    return filtered
```

#### **Groq Llama-70b Output:**

from typing import List

def deadband_filter(readings: List[float], deadband: float) -> List[float]:
    """
    Implements deadband filtering on a list of readings.
    Readings within [-deadband, +deadband] are considered 'dead zone' and are filtered out.
    """
    # Input validation
    if not isinstance(readings, list):
        raise TypeError("readings must be a list")
    if not isinstance(deadband, (int, float)):
        raise TypeError("deadband must be a number")
    if deadband < 0:
        raise ValueError("deadband must be non-negative")

    result = []
    for reading in readings:
        if abs(reading) > deadband:
            result.append(reading)
    return result

*   **Robustness:** Both validate `deadband >= 0`. Groq adds extra type checks for the `readings` input list itself.
*   **Verdict:** DeepSeek's output is more aligned with fleet terminology. Groq's is slightly more defensive on input types.

### **Function 2: `parse_fleet_log`**
**Prompt:** "Write a Python function `parse_fleet_log(log_line: str) -> dict`. A fleet log line format is: `'2026-04-20 01:23:45 [AGENT] [CYCLE 137] STATUS: message'`. Parse it into a dict with keys: `timestamp`, `agent`, `cycle`, `status`, `message`. Assume the format is consistent but be defensive against minor variations."

#### **DeepSeek-chat (Self) Output:**
```python
import re
from typing import Optional, Dict

def parse_fleet_log(log_line: str) -> Dict[str, Optional[str]]:
    """
    Parse a structured fleet log line into its components.

    Expected format: 'YYYY-MM-DD HH:MM:SS [AGENT] [CYCLE N] STATUS: message'
    Returns a dictionary with keys: timestamp, agent, cycle, status, message.
    Returns None values for keys if parsing fails for that component.
    """
    pattern = r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) \[([^\]]+)\] \[CYCLE (\d+)\] ([A-Z]+): (.+)$'
    match = re.match(pattern, log_line.strip())

    if not match:
        # Fallback: return a dict with None values
        return {key: None for key in ["timestamp", "agent", "cycle", "status", "message"]}

    timestamp, agent, cycle, status, message = match.groups()
    return {
        "timestamp": timestamp,
        "agent": agent,
        "cycle": cycle,
        "status": status,
        "message": message.strip()
    }
```
