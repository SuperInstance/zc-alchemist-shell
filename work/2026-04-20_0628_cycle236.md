## **2. Function 1: `deadband_filter`**

**Prompt to both models:**
```
Write a Python function called `deadband_filter` that implements the Cocapn fleet's Deadband Protocol.

The function must:
- Take two arguments: `readings` (list of floats) and `deadband` (float, default=0.1).
- Return a filtered list of floats.
- Implement the protocol logic:
  P0 (Negative Space): Identify readings that are within +/- `deadband` of zero. These are "dead zone" readings to be removed.
  P1 (Safe Channels): From the remaining readings, identify those with absolute value > `deadband`. These are valid signals.
  P2 (Optimize): (Optional for this function) Could be calculating a rolling average, but for now, just return the P1 list.
- Include a clear docstring explaining the P0/P1 logic.
- Include type hints.
- Include basic error handling (e.g., for empty list).
```

### **2.1 DeepSeek-chat Output**
```python
from typing import List

def deadband_filter(readings: List[float], deadband: float = 0.1) -> List[float]:
    """
    Apply the Cocapn fleet Deadband Protocol to sensor readings.

    Protocol:
    P0 (Map negative space): Remove readings within +/- `deadband` of zero (dead zone).
    P1 (Find safe channels): Keep readings where absolute value > `deadband` (valid signals).

    Args:
        readings: List of floating-point sensor readings.
        deadband: Threshold for dead zone (default 0.1).

    Returns:
        Filtered list of readings that are outside the deadband.

    Raises:
        ValueError: If `readings` is empty.
    """
    if not readings:
        raise ValueError("Readings list cannot be empty")

    # P0: Identify and exclude negative space (dead zone)
    # P1: Identify safe channels (valid signals)
    filtered = [
        reading for reading in readings
        if abs(reading) > deadband  # Combined P0 exclusion + P1 inclusion
    ]

    return filtered
```

from typing import Dict, List
import re

def parse_fleet_context(markdown_text: str) -> Dict[str, List[str] | str]:
    """
    Parse a FLEET-CONTEXT.md style markdown document into structured data.

    Args:
        markdown_text: The full markdown text of the fleet context document.

    Returns:
        Dictionary with keys:
        - 'agents': List of agent names/descriptions.
        - 'technologies': List of technology names.
        - 'rules': List of fleet rules.
        - 'doctrine': The doctrine description string.
    """
    result = {
        'agents': [],
        'technologies': [],
        'rules': [],
        'doctrine': ''
    }

    lines = markdown_text.split('\n')
    current_section = None

    for line in lines:
        # Detect section headers
        if line.startswith('## Who We Are'):
            current_section = 'agents'
            continue
        elif line.startswith('## Key Technologies'):
            current_section = 'technologies'
            continue
        elif line.startswith('## Rules'):
            current_section = 'rules'
            continue
        elif line.startswith('## Fleet Doctrine'):
            current_section = 'doctrine'
            continue
        elif line.startswith('## '):  # Other section, stop capturing
            current_section = None
