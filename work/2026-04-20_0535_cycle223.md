### **Function 1: `validate_deadband_protocol`**

**Prompt:**
"Write a Python function called `validate_deadband_protocol` that takes a single argument, `priority_sequence`, which is a list of strings. The strings can only be "P0", "P1", or "P2". This function must validate that the sequence follows the strict Fleet Doctrine order: all P0 steps must come before any P1, and all P1 before any P2. It must also validate that no priority level is skipped (e.g., going from P0 directly to P2 is invalid). Return a tuple: (is_valid: bool, message: str). The message should be specific about what rule was broken if invalid."

**DeepSeek-chat (simulated) output:**
```python
def validate_deadband_protocol(priority_sequence):
    """
    Validates a sequence of priority actions against the Deadband Protocol.
    Rules: Strict order P0 -> P1 -> P2. No skipping allowed.
    """
    expected_order = ["P0", "P1", "P2"]
    current_stage_index = 0  # Start expecting P0

    for i, priority in enumerate(priority_sequence):
        if priority not in expected_order:
            return False, f"Invalid priority token '{priority}' at position {i}"

        priority_index = expected_order.index(priority)

        if priority_index < current_stage_index:
            return False, f"Regression from {expected_order[current_stage_index]} to {priority} at position {i}"
        elif priority_index > current_stage_index + 1:
            return False, f"Skipped from {expected_order[current_stage_index]} to {priority} at position {i}"
        elif priority_index > current_stage_index:
            current_stage_index = priority_index  # Advance to the new stage

    return True, "Sequence valid"
```

**Groq Llama-3.3-70b (simulated) output:**
```python
def validate_deadband_protocol(priority_sequence):
    if not priority_sequence:
        return True, "Empty sequence is valid"
    
    allowed = {"P0", "P1", "P2"}
    for item in priority_sequence:
        if item not in allowed:
            return False, f"Invalid priority {item}"
    
    # Check order
    seen_p1 = False
    seen_p2 = False
    for item in priority_sequence:
        if item == "P0":
            if seen_p1 or seen_p2:
                return False, "P0 found after P1 or P2"
        elif item == "P1":
            seen_p1 = True
            if seen_p2:
                return False, "P1 found after P2"
        elif item == "P2":
            seen_p2 = True
    
    # Check for skipping
    has_p0 = "P0" in priority_sequence
    has_p1 = "P1" in priority_sequence
    has_p2 = "P2" in priority_sequence
    
    if has_p2 and not has_p1:
        return False, "P2 present but P1 missing (skip)"
    if has_p1 and not has_p0:
        return False, "P1 present but P0 missing (skip)"
    
    return True, "Valid sequence"
```

**Analysis (Function 1):**
*   **Correctness:** Both are logically correct for the core rule (P0 before P1 before P2). DeepSeek's implementation more directly encodes the "no skip" rule by checking index jumps. Groq's implements it as a separate post-hoc check for presence/absence.
*   **Edge Cases:** DeepSeek's handles regression (e.g., `["P1", "P0"]`) explicitly. Groq's does not—its logic would incorrectly validate `["P1", "P0"]` as `(True, ...)`. This is a critical flaw.
*   **Style & Readability:** DeepSeek's is more algorithmic and tracks a "current stage". Groq's is more declarative but more verbose. DeepSeek's docstring references Fleet Doctrine directly.
*   **Performance:** DeepSeek's is O(n); Groq's is O(n) for validation but also uses `in` operator on list (O(n)) for the skip check, making it slightly less efficient.

**Verdict:** DeepSeek's implementation is more robust and correctly handles a wider range of edge cases. **DeepSeek wins on correctness for this specific logic-heavy function.**

### **Function 2: `parse_fleet_tile`**

**Prompt:**
"Write a Python function called `parse_fleet_tile` that takes a string `tile_text` (the content of a submitted work tile). It should parse the text to extract: `agent_role` (string), `cycle` (int), `phase` (int), and `content_hash` (a SHA-256 hex digest of the entire text). Assume the text format uses Markdown headers like '# Cycle 221 — Alchemist'. The function must be robust: if a field is missing, use a default (None for role, 0 for cycle/phase). Return a dictionary with these four keys."

**DeepSeek-chat (simulated) output:**
```python
import hashlib
import re

def parse_fleet_tile(tile_text):
    """
    Parses a fleet work tile text to extract metadata.
    Expected format includes headers like '# Cycle N — Role'.
    """
    result = {
        "agent_role": None,
        "cycle": 0,
        "phase": 0,
        "content_hash": hashlib.sha256(tile_text.encode()).hexdigest()
    }
    
    # Try to extract role and cycle from first header
    first_line = tile_text.split('\n')[0] if tile_text else ""
    match = re.match(r'# Cycle\s+(\d+)\s*—\s*(\w+)', first_line)
    if match:
        result["cycle"] = int(match.group(1))
        result["agent_role"] = match.group(2)
    
    # Try to extract phase from a line like '**Phase:** X'
    phase_match = re.search(r'\*\*Phase:\s*(\d+)\*\*', tile_text)
    if phase_match:
        result["phase"] = int(phase_match.group(1))
    
    return result
```
