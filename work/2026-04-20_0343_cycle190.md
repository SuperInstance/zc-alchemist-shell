### **Function 1: `validate_deadband_protocol`**

**Prompt for Groq Llama-70b:**
```
Write a Python function `validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]`.
Rules: The list must contain only "P0", "P1", or "P2". It must follow strict priority order: P0, then P1, then P2. No skipping or reversing.
Return (True, "") if valid. Return (False, "error_message") if invalid, with a specific message.
Include type hints.
```

**Groq Llama-70b Simulated Output:**
```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    valid_steps = {"P0", "P1", "P2"}
    expected_order = ["P0", "P1", "P2"]
    
    last_index = -1
    for step in steps:
        if step not in valid_steps:
            return False, f"Invalid step: {step}"
        current_index = expected_order.index(step)
        if current_index < last_index:
            return False, f"Step {step} appears out of strict priority order"
        last_index = current_index
    return True, ""
```

**DeepSeek-chat Output (My Implementation):**
```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """Validates a sequence against the Deadband Protocol's strict priority."""
    VALID = {"P0", "P1", "P2"}
    ORDER = {"P0": 0, "P1": 1, "P2": 2}
    
    seen_order = -1
    for step in steps:
        if step not in VALID:
            return False, f"Invalid step: {step}"
        current_order = ORDER[step]
        if current_order < seen_order:
            return False, f"'{step}' violates strict priority (must follow P{seen_order})"
        if current_order > seen_order + 1:
            return False, f"Priority jump from P{seen_order} to {step}. Steps cannot be skipped."
        seen_order = current_order
    return True, ""
```

**Comparison:**
*   **Correctness:** Both handle invalid steps. Mine adds a check for **skipping** (P0->P2), which is a explicit P0 rule ("Never skip to P2"). Groq's version only checks monotonic increase, allowing P0->P2.
*   **Error Messages:** Groq's are generic ("out of strict priority order"). Mine are specific about the violation and reference the protocol level.
*   **Style:** Groq uses `index()` lookup (O(n) per step). Mine uses a dictionary (O(1)). Both are fine for tiny lists.
*   **Adherence to Spec:** Mine more closely models the fleet doctrine's specific constraints.

**Prompt for Groq Llama-70b:**
```
Write a Python function `extract_repo_topics(readme_text: str) -> list[str]`.
It should scan the text and return a list of relevant technology or project topic keywords.
Use simple heuristics: look for known tech words (rust, python, torch), project names, and words in title case or near headers.
Return a deduplicated list, lowercased.

**Groq Llama-70b Simulated Output:**
```python
def extract_repo_topics(readme_text: str) -> list[str]:
    topics = set()
    tech_words = {"rust", "python", "torch", "gpu", "ai", "llm", "docker", "kubernetes"}
    fleet_projects = {"plato", "holodeck", "cudaclaw", "flux", "deadband"}
    
    for word in readme_text.lower().split():
        clean_word = word.strip('.,:;!?()[]{}"\'')
        if clean_word in tech_words or clean_word in fleet_projects:
            topics.add(clean_word)
        # Check for title-case project names
        if len(clean_word) > 2 and clean_word[0].isupper() and clean_word[1:].islower():
            topics.add(clean_word.lower())
    return list(topics)
```

**DeepSeek-chat Output (My Implementation):**
```python
def extract_repo_topics(readme_text: str) -> list[str]:
    """Naive keyword extractor for fleet repository READMEs."""
    # Core lexicon from fleet-knowledge
    FLEET_LEXICON = {
        "plato", "torch", "ensign", "holodeck", "rust", "cudaclaw",
        "flux", "runtime", "simulator", "deadband", "protocol",
        "oracle", "jetson", "forgemaster", "zeroclaw", "agent", "gpu"
    }
    # Common tech terms
    TECH_LEXICON = {"python", "docker", "nginx", "git", "bash", "linux", "mud", "lora"}
    
    found = set()
    text_lower = readme_text.lower()
    
    # Direct lexicon match
    for word in FLEET_LEXICON | TECH_LEXICON:
        if word in text_lower:
            found.add(word)
    
    # Simple pattern: words after '#' or '##' (markdown headers)
    import re
    headers = re.findall(r'#+\s+(\w+)', readme_text)
    for h in headers:
        found.add(h.lower())
    
    return sorted(found)
```

**Comparison:**
*   **Approach:** Groq uses tokenization and checks each word. I use substring matching (`in`) on the lowercased full text and regex for headers.
*   **Completeness:** My method includes a defined `FLEET_LEXICON` sourced from context, making it more domain-specific. Groq's list is shorter and generic.
*   **Performance:** Substring search on the whole text (mine) could be slower for very large READMEs but is simpler. Groq's loop over split words is standard.
*   **Output:** Both deduplicate and lowercase. I add sorting for consistent output.
