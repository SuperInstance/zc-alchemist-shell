### **Function 1: `validate_deadband_protocol`**

**DeepSeek-chat Implementation:**
```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """Validate steps against P0->P1->P2 priority."""
    priorities = {"P0": 0, "P1": 1, "P2": 2}
    last_priority = -1
    
    for i, step in enumerate(steps):
        # Extract priority prefix
        if not step.startswith(("P0", "P1", "P2")):
            return False, f"Step {i}: '{step}' must start with P0, P1, or P2"
        
        prefix = step[:2]
        current_priority = priorities.get(prefix, -1)
        
        # Check for non-decreasing priority
        if current_priority < last_priority:
            return False, f"Step {i}: '{step}' violates P0->P1->P2 order"
        
        last_priority = current_priority
    
    return True, "Valid Deadband Protocol sequence"
```

**Groq Llama-3.3-70b-Specdec Implementation:**
```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """Check if steps follow P0->P1->P2 strict priority."""
    expected_order = ["P0", "P1", "P2"]
    seen_priority = None
    
    for idx, step in enumerate(steps):
        # Get first two chars as priority tag
        if len(step) < 2:
            return False, f"Step {idx} too short: '{step}'"
        
        priority = step[:2]
        if priority not in expected_order:
            return False, f"Step {idx}: invalid priority '{priority}' in '{step}'"
        
        # Convert to numeric for comparison
        priority_idx = expected_order.index(priority)
        if seen_priority is not None and priority_idx < seen_priority:
            return False, f"Step {idx}: '{step}' breaks order (went back to {priority})"
        
        # Update last seen priority
        if seen_priority is None or priority_idx > seen_priority:
            seen_priority = priority_idx
    
    return True, "Protocol valid"
```

**Comparison:**
*   **Correctness:** Both correctly validate the sequence. DeepSeek uses a dict mapping, Groq uses `index()` on a list.
*   **Conciseness:** DeepSeek's version is slightly more compact (9 lines vs 11 lines of core logic).
*   **Spec Adherence:** Both match the signature. DeepSeek's error messages are more specific ("must start with...").
*   **Robustness:** Groq explicitly checks `len(step) < 2`, which DeepSeek misses. DeepSeek's `startswith(("P0", "P1", "P2"))` is elegant but doesn't catch malformed short strings like `"P"`.

**Edge Case Test:** `steps = ["P0", "P", "P2"]`
*   DeepSeek: Returns `(False, "Step 1: 'P' must start with P0, P1, or P2")` ✓
*   Groq: Returns `(False, "Step 1 too short: 'P'")` ✓

### **Function 2: `extract_training_tiles`**

**DeepSeek-chat Implementation:**
```python
def extract_training_tiles(log_text: str) -> list[dict]:
    """Extract training tiles from agent work log."""
    tiles = []
    lines = log_text.split('\n')
    in_tile = False
    current_tile = {"id": "", "content": [], "agent": ""}
    
    for line in lines:
        if line.startswith('## Tile:'):
            if in_tile:
                # Save previous tile
                current_tile["content"] = '\n'.join(current_tile["content"]).strip()
                tiles.append(current_tile.copy())
            
            # Start new tile
            in_tile = True
            current_tile = {"id": "", "content": [], "agent": ""}
            
            # Parse tile header
            parts = line[8:].strip().split('|')
            if len(parts) >= 2:
                current_tile["id"] = parts[0].strip()
                current_tile["agent"] = parts[1].strip() if len(parts) > 1 else ""
        
        elif line.startswith('## ') and in_tile:
            # End of current tile
            current_tile["content"] = '\n'.join(current_tile["content"]).strip()
            tiles.append(current_tile.copy())
            in_tile = False
            current_tile = {"id": "", "content": [], "agent": ""}
        
        elif in_tile and line != "":
            current_tile["content"].append(line)
    
    # Capture last tile if log ends without another ##
    if in_tile and current_tile["content"]:
        current_tile["content"] = '\n'.join(current_tile["content"]).strip()
        tiles.append(current_tile)
    
    return tiles
```

**Groq Llama-3.3-70b-Specdec Implementation:**
```python
def extract_training_tiles(log_text: str) -> list[dict]:
    """Parse log text to extract training tiles."""
    import re
    
    tiles = []
    # Pattern to match tile headers: ## Tile: ID | Agent
    tile_pattern = r'^## Tile:\s*(.+?)\s*\|\s*(.+?)$'
    
    lines = log_text.split('\n')
    i = 0
    while i < len(lines):
        match = re.match(tile_pattern, lines[i])
        if match:
            tile_id, agent = match.groups()
            content_lines = []
            i += 1
            
            # Collect content until next ## or end
            while i < len(lines) and not lines[i].startswith('## '):
                content_lines.append(lines[i])
                i += 1
            
            tile = {
                "id": tile_id.strip(),
                "content": '\n'.join(content_lines).strip(),
                "agent": agent.strip()
            }
            tiles.append(tile)
        else:
            i += 1
    
    return tiles
```

**Comparison:**
*   **Correctness:** Both will extract tiles. DeepSeek uses state-machine parsing, Groq uses regex.
*   **Conciseness:** Groq's regex approach is more concise (15 lines vs 25 lines).
*   **Spec Adherence:** DeepSeek's output matches `list[dict]` exactly. Groq's does too.
*   **Robustness:** DeepSeek handles empty lines within content better (skips `line != ""`). Groq's regex might fail if the header format varies (e.g., missing agent). DeepSeek's parsing is more tolerant of format variations.
