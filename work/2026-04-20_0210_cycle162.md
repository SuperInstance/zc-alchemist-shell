## **2. Function 1: `validate_deadband_protocol`**

**Prompt:**
```
Write a Python function `validate_deadband_protocol` that analyzes a list of action dictionaries. Each dict has keys 'phase' ('P0', 'P1', 'P2') and 'action' (str). The Deadband Protocol requires strict order: all P0, then all P1, then all P2. The function must return a dict with 'is_valid' (bool) and 'violations' (list of strings describing any out-of-order actions). Include type hints and a clear docstring.
```

### **DeepSeek-chat Implementation:**
```python
from typing import List, Dict, Any, Literal

def validate_deadband_protocol(action_sequence: List[Dict[str, Any]]) -> Dict[str, Any]:
    """
    Validates a sequence of actions against the Deadband Protocol's strict phase priority.

    The protocol mandates: P0 (map negative space) -> P1 (find safe channels) -> P2 (optimize).
    Actions must appear in non-decreasing phase order.

    Args:
        action_sequence: List of dictionaries, each with 'phase' ('P0','P1','P2') and 'action' keys.

    Returns:
        Dictionary with:
            - 'is_valid': True if sequence adheres to protocol, False otherwise.
            - 'violations': List of descriptive violation messages, empty if valid.
    """
    expected_order = ['P0', 'P1', 'P2']
    current_phase_index = 0
    violations = []

    for i, action in enumerate(action_sequence):
        phase = action.get('phase')
        action_desc = action.get('action', f'action_{i}')

        if phase not in expected_order:
            violations.append(f"Action '{action_desc}' has invalid phase '{phase}'. Must be P0, P1, or P2.")
            continue

        phase_index = expected_order.index(phase)

        if phase_index < current_phase_index:
            violations.append(
                f"Protocol violation at position {i}: '{action_desc}' (phase {phase}) "
                f"appears after a higher phase ({expected_order[current_phase_index]})."
            )
        # Only advance the expected phase index if we encounter a *new* higher phase
        elif phase_index > current_phase_index:
            current_phase_index = phase_index

    return {
        'is_valid': len(violations) == 0,
        'violations': violations
    }
```

    """

    Parameters:
    actions: list of dicts with keys 'phase' and 'action'

    Returns:
    dict with 'valid' (bool) and 'errors' (list)
    """
    order = {"P0": 0, "P1": 1, "P2": 2}
    seen_phase = None
    errors = []

    for idx, act in enumerate(actions):
        phase = act.get("phase")
        action_name = act.get("action", f"#{idx}")

        if phase not in order:
            errors.append(f"Invalid phase '{phase}' for action '{action_name}'.")
            continue

        if seen_phase is None:
            seen_phase = phase
        else:
            if order[phase] < order[seen_phase]:
                errors.append(
                    f"Order violation: {phase} (action '{action_name}') came after {seen_phase}."
                )
            seen_phase = phase if order[phase] > order[seen_phase] else seen_phase

    return {"is_valid": len(errors) == 0, "violations": errors}

*   **Clarity & Style:** DeepSeek uses more descriptive names, a full docstring with Args/Returns, and `Literal` for type hints. Groq's is more concise, with a simpler docstring and use of `Union` for the return type.

*   **Edge Cases:** DeepSeek handles invalid phases within the loop, continuing validation. Groq does similarly. Both would fail silently on missing 'phase' key (returning `None`).
