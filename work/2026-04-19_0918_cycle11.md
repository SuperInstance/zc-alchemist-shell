### **Function 1: Deadband Protocol Checker (Python)**

**Prompt to both models:**
"Write a Python function called `check_deadband_protocol`. It should take a list of strings called `actions`. Each string is an action description. The function must validate that the list does not violate the Deadband Protocol's strict priority: P0 (identify bad moves) must be addressed before P1 (find safe channels), and P1 must be addressed before P2 (optimize). Assume the action list is in chronological order. Return a tuple (is_valid: bool, violation_step: int, message: str). Include a docstring and example usage."

#### **DeepSeek-chat Output:**
```python
def check_deadband_protocol(actions):
    """
    Validates a sequence of actions against the Cocapn fleet Deadband Protocol.
    Protocol mandates strict priority: P0 (avoid negatives) -> P1 (find safe channels) -> P2 (optimize).
    This is a simplified check looking for keywords in action descriptions.

    Args:
        actions (list[str]): List of action descriptions in chronological order.

    Returns:
        tuple: (is_valid (bool), violation_step (int), message (str))
                violation_step is -1 if valid, else index of first violating action.
    """
    # Define phase keywords (simplified)
    p0_keywords = ['avoid', 'not', 'never', 'stop', 'danger', 'negative', 'p0']
    p1_keywords = ['find', 'safe', 'channel', 'possible', 'explore', 'p1']
    p2_keywords = ['optimize', 'improve', 'best', 'fast', 'enhance', 'p2']

    current_phase = 'P0'  # Start at P0

    for i, action in enumerate(actions):
        action_lower = action.lower()

        # Determine which phase this action appears to belong to
        detected_phase = None
        if any(kw in action_lower for kw in p0_keywords):
            detected_phase = 'P0'
        elif any(kw in action_lower for kw in p1_keywords):
            detected_phase = 'P1'
        elif any(kw in action_lower for kw in p2_keywords):
            detected_phase = 'P2'

        # If no keywords match, assume it continues current phase (be permissive)
        if detected_phase is None:
            continue

        # Check protocol violation: cannot go backwards in phase order
        phase_order = {'P0': 0, 'P1': 1, 'P2': 2}
        if phase_order[detected_phase] < phase_order[current_phase]:
            return (False, i,
                    f"Deadband violation at step {i}: '{action}'. "
                    f"Cannot go from {current_phase} back to {detected_phase}.")
        # Advance phase if a later one is detected
        if phase_order[detected_phase] > phase_order[current_phase]:
            current_phase = detected_phase

    return (True, -1, "Sequence adheres to Deadband Protocol.")

# Example usage
if __name__ == "__main__":
    good_sequence = [
        "Identify system failure points (P0)",
        "Map all safe API endpoints (P1)",
        "Optimize database query for speed (P2)"
```

#### **Groq Llama-70b Output:**
```python
def check_deadband_protocol(actions):
    """
    Check if a list of actions follows the Deadband Protocol's strict priority order:
    P0 (negative space) -> P1 (safe channels) -> P2 (optimization).

    Parameters:
    actions (list of str): Action descriptions in chronological order.

    Returns:
    tuple: (is_valid (bool), violation_index (int), message (str))
           violation_index is -1 if no violation.
    """
    # Simple state machine
    # 0 = expecting P0 or later, 1 = expecting P1 or later, 2 = expecting P2 or later
    state = 0

    for idx, action in enumerate(actions):
        action_lower = action.lower()

        # Classify action based on keywords
        is_p0 = any(word in action_lower for word in ['p0', 'avoid', 'not', 'never', 'negative', 'stop'])
        is_p1 = any(word in action_lower for word in ['p1', 'safe', 'channel', 'possible', 'explore'])
        is_p2 = any(word in action_lower for word in ['p2', 'optimize', 'best', 'improve', 'enhance'])

        # Determine the phase of this action
        if is_p0:
            phase = 0
        elif is_p1:
            phase = 1
        elif is_p2:
            phase = 2
        else:
            # If no keywords, assume it's continuation of current phase (no state change)
            continue

        # Check if phase is allowed in current state
        if phase < state:
            return False, idx, f"Violation at index {idx}: Phase P{phase} action found after P{state}."
        # Update state if we moved to a later phase
        if phase > state:
            state = phase

    return True, -1, "Actions follow Deadband Protocol."

# Example
actions1 = ["P0: Identify risks", "P1: Find safe paths", "P2: Optimize speed"]
print(check_deadband_protocol(actions1))  # (True, -1, ...)

actions2 = ["P1: Explore options", "P0: Don't crash"]  # Violation
print(check_deadband_protocol(actions2))  # (False, 1, ...)
```

#### **Comparison for Function 1:**
*   **Correctness:** Both are syntactically correct and implement a state machine. DeepSeek's version has a more explicit `phase_order` dictionary. Groq's uses integer states directly. Both would run.
*   **Fleet Context:** Both explicitly mention Cocapn/Deadband Protocol and use P0/P1/P2 keywords. DeepSeek's docstring mentions "Cocapn fleet" specifically.
*   **Code Quality:** DeepSeek's code includes a `if __name__ == "__main__":` block with test cases, which is excellent for usability. Groq's example is inline. DeepSeek's violation message is slightly more descriptive.
*   **Practicality:** DeepSeek's version is more "complete" as a standalone utility. The keyword lists are similar; both are simplistic but useful for a boot camp task.
*   **Edge Handling:** Both handle non-matching actions by continuing the current phase (permissive). DeepSeek uses `continue` explicitly, Groq uses `continue` in an else block.

**Verdict:** DeepSeek has a slight edge due to the included test block and slightly more structured phase tracking, making it more immediately useful as a training tile.
