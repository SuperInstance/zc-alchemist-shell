## **3. Function 1: deadband_filter**

**Spec:**
```
Implement a deadband filter for sensor data.

Input: values (list of floats), deadband (float)
Output: filtered list of floats

P0 (negative space): Remove values where abs(value) < deadband (too close to zero).
P1 (safe channels): Keep values where deadband <= abs(value) <= 2*deadband, but replace with sign(value)*deadband.
P2 (optimize): For abs(value) > 2*deadband, keep original.

Example:
values = [-3.0, -0.5, 0.1, 1.5, 2.5, 4.0], deadband=1.0
-> [-3.0, -1.0, 0.0, 1.0, 2.5, 4.0]
```

### **DeepSeek-chat output:**
```python
from typing import List

def deadband_filter(values: List[float], deadband: float) -> List[float]:
    """
    Apply a deadband filter to a list of float values based on the Deadband Protocol.

    Parameters:
    values: List of input float values
    deadband: Positive float defining the deadband region

    Returns:
    Filtered list where:
    - P0: |value| < deadband -> 0.0
    - P1: deadband <= |value| <= 2*deadband -> sign(value) * deadband
    - P2: |value| > 2*deadband -> original value
    """
    if deadband <= 0:
        raise ValueError("deadband must be positive")

    result = []
    for v in values:
        abs_v = abs(v)
        if abs_v < deadband:
            result.append(0.0)
        elif abs_v <= 2 * deadband:
            result.append((1.0 if v >= 0 else -1.0) * deadband)
        else:
            result.append(v)
    return result
```

### **Groq Llama-3.3-70b output:**
```python
from typing import List

def deadband_filter(values: List[float], deadband: float) -> List[float]:
    """
    Apply a deadband filter to sensor data.

    Args:
        values: A list of floating-point sensor readings.
        deadband: The deadband threshold (positive float).

    Returns:
        A filtered list where values within the deadband are set to zero,
        values within [deadband, 2*deadband] are clamped to +/- deadband,
        and values beyond 2*deadband are passed through.
    """
    if deadband <= 0:
        raise ValueError("Deadband must be positive.")

    filtered = []
    for val in values:
        abs_val = abs(val)
        if abs_val < deadband:
            filtered.append(0.0)
        elif abs_val <= 2 * deadband:
            # Clamp to +/- deadband
            filtered.append(deadband if val > 0 else -deadband)
        else:
            filtered.append(val)
    return filtered
```

**Comparison:**
- Both correct, handle deadband <= 0.
- DeepSeek adds explicit sign handling `(1.0 if v >= 0 else -1.0)`; Groq uses ternary `deadband if val > 0 else -deadband`. Edge: val == 0? Already caught by P0.
- DeepSeek docstring references P0/P1/P2 explicitly — better for fleet context.
- Groq docstring slightly more generic.
- Both type-hinted, readable.

**Winner:** DeepSeek (slightly better fleet alignment).

## **4. Function 2: parse_fleet_log**

**Spec:**
```
Parse a fleet log line format: "YYYY-MM-DD HH:MM:SS | AGENT | LEVEL | message"

Input: log_line (string)
Output: dict with keys: timestamp (datetime), agent (str), level (str), message (str)

If line doesn't match format, return None.
Use datetime.strptime for parsing.
```

### **DeepSeek-chat output:**
```python
from datetime import datetime
from typing import Optional, Dict

def parse_fleet_log(log_line: str) -> Optional[Dict[str, any]]:
    """
    Parse a structured fleet log line into its components.

    Expected format: "YYYY-MM-DD HH:MM:SS | AGENT | LEVEL | message"

    Args:
        log_line: A string log line.

    Returns:
        Dictionary with keys: timestamp (datetime), agent (str), level (str), message (str)
        Returns None if format doesn't match.
    """
    parts = log_line.split(' | ', maxsplit=3)
    if len(parts) != 4:
        return None

    date_str, agent, level, message = parts
    try:
        timestamp = datetime.strptime(date_str, '%Y-%m-%d %H:%M:%S')
    except ValueError:
        return None

    return {
        'timestamp': timestamp,
        'agent': agent.strip(),
        'level': level.strip(),
        'message': message.strip()
    }
```

### **Groq Llama-3.3-70b output:**

```python
from typing import Optional, Dict

def parse_fleet_log(log_line: str) -> Optional[Dict[str, any]]:
    """
    Parses a fleet log line in the format "YYYY-MM-DD HH:MM:SS | AGENT | LEVEL | message".

    Args:
        log_line: The log line string to parse.

    Returns:
        A dictionary with keys 'timestamp' (datetime), 'agent' (str), 'level' (str), 'message' (str),
        or None if the line does not match the expected format.
    """
    # Split into exactly four parts
    segments = log_line.split(' | ', 3)
    if len(segments) != 4:
        return None

    date_time_str, agent, level, message = segments

    # Parse the timestamp
    try:
        timestamp = datetime.datetime.strptime(date_time_str, '%Y-%m-%d %H:%M:%S')
    except ValueError:
        return None

    # Return structured data
    return {
        'timestamp': timestamp,
        'agent': agent.strip(),
        'level': level.strip(),
        'message': message.strip()
    }
```

**Comparison:**
- Nearly identical. Both use `split(' | ', maxsplit=3)` (Groq uses `3`, DeepSeek `maxsplit=3`).
- Both handle ValueError on datetime parse.
- Both strip() fields.
- DeepSeek uses `from datetime import datetime`; Groq uses `import datetime`. Both fine.
- DeepSeek's docstring says "Returns None if format doesn't match" more explicitly.

**Winner:** Tie.
