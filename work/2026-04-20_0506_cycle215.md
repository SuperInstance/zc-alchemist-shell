## **2. Function A: `parse_fleet_log`**

**Prompt:**
```
Write a Python function named `parse_fleet_log` that parses a structured fleet log line into a dictionary.

Specifications:
- Input: A string like "2026-04-20 05:15:23 | AGENT Alchemist | CYCLE 215 | ACTION tile_submit | STATUS success"
- Output: A dict with keys: timestamp, agent, cycle, action, status.
- The timestamp should be a datetime object.
- Cycle should be an integer.
- If the line doesn't match the expected format, return None.
- Use robust parsing, not brittle splits.

Return only the function code, no explanations.
```

### **DeepSeek-chat Output:**
```python
import re
from datetime import datetime

def parse_fleet_log(log_line):
    pattern = r'(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) \| AGENT (\w+) \| CYCLE (\d+) \| ACTION (\w+) \| STATUS (\w+)'
    match = re.match(pattern, log_line)
    if not match:
        return None
    timestamp_str, agent, cycle_str, action, status = match.groups()
    try:
        timestamp = datetime.strptime(timestamp_str, '%Y-%m-%d %H:%M:%S')
        cycle = int(cycle_str)
    except ValueError:
        return None
    return {
        'timestamp': timestamp,
        'agent': agent,
        'cycle': cycle,
        'action': action,
        'status': status
    }
```

### **Groq Llama-70b Output:**

### **Comparison for Function A:**

- **Clarity:** Both are clear. Groq's comment is helpful but not necessary.
- **Edge Cases:** Both handle format mismatch and value errors. Groq explicitly strips whitespace, which is robust.
- **Verdict:** Tie. Groq's version is slightly more defensive; DeepSeek's is slightly leaner.

## **3. Function B: `validate_deadband_protocol`**

**Prompt:**
```
Write a Python function named `validate_deadband_protocol` that validates a sequence of actions against P0/P1/P2 priorities.

Specifications:
- Input: A list of strings, each string is an action like "P0_map_negative", "P1_find_channel", "P2_optimize".
- The Deadband Protocol requires strict priority order: all P0 before any P1, all P1 before any P2.
- Return a tuple (is_valid, error_message).
- If valid, error_message is empty string.
- If invalid, error_message explains the violation.
- Assume actions are prefixed exactly with "P0_", "P1_", or "P2_".

Return only the function code, no explanations.
```

### **DeepSeek-chat Output:**
```python
def validate_deadband_protocol(actions):
    expected_order = ['P0', 'P1', 'P2']
    current_phase = 0
    for i, action in enumerate(actions):
        if not action.startswith(('P0_', 'P1_', 'P2_')):
            return False, f"Action '{action}' at index {i} has invalid prefix."
        action_phase = action[:2]
        phase_index = expected_order.index(action_phase)
        if phase_index < current_phase:
            return False, f"Action '{action}' at index {i} violates order: {expected_order[current_phase]} expected."
        current_phase = phase_index
    return True, ""
```

### **Groq Llama-70b Output:**
```python
def validate_deadband_protocol(actions):
    if not isinstance(actions, list):
        return False, "Input must be a list"
    allowed_prefixes = ['P0_', 'P1_', 'P2_']
    last_prefix = None
    for idx, action in enumerate(actions):
        if not any(action.startswith(prefix) for prefix in allowed_prefixes):
            return False, f"Action '{action}' at index {idx} has invalid prefix."
        prefix = action[:3]  # Get 'P0_', 'P1_', or 'P2_'
        if last_prefix is None:
            last_prefix = prefix
        else:
            # Check order: P0_ must come before P1_, P1_ before P2_
            if (prefix == 'P0_' and last_prefix in ['P1_', 'P2_']) or (prefix == 'P1_' and last_prefix == 'P2_'):
                return False, f"Priority violation at index {idx}: '{action}' appears after higher priority actions."
            if prefix != last_prefix:
                last_prefix = prefix
    return True, ""
```

### **Comparison for Function B:**
- **Correctness:** Both are logically correct, but Groq's has a bug: `prefix = action[:3]` gets `'P0_'` but later comparisons use `'P0_'` vs `'P1_'` correctly. However, the condition `(prefix == 'P0_' and last_prefix in ['P1_', 'P2_'])` is reversed—it should catch P0 after P1/P2, but the description says "P0_ must come before P1_", which is correct. Wait, the condition is actually checking for P0 appearing after P1/P2, which is correct. But the logic is convoluted. DeepSeek's phase tracking is simpler and correct.
- **Conciseness:** DeepSeek is more concise (9 lines vs 15).
- **Clarity:** DeepSeek's algorithm is clearer: track current allowed phase. Groq's nested condition is harder to follow.
- **Edge Cases:** Groq checks `isinstance(actions, list)`, which is extra defensive. DeepSeek assumes list.
- **Verdict:** DeepSeek's version is superior—simpler, correct, and easier to maintain.
