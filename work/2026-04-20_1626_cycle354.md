## **2. Function 1: Deadband Filter**

**Prompt to both models:**
```
Write a Python function `apply_deadband_protocol` that takes a list of action dictionaries. Each dict has keys "name" (str), "risk_score" (int 0-10), and "potential_reward" (int 0-10). Implement the fleet's Deadband Protocol: P0 actions (risk_score >= 8) are removed entirely. P1 actions (risk_score 4-7) are kept but marked as "channel". P2 actions (risk_score 0-3) are kept and marked as "optimize". Return a filtered list with added "protocol_phase" key.
```

### **DeepSeek-chat output:**
```python
def apply_deadband_protocol(actions):
    """
    Apply fleet Deadband Protocol (P0/P1/P2) to action list.
    
    Args:
        actions: List of dicts with keys "name", "risk_score", "potential_reward"
    
    Returns:
        Filtered list with added "protocol_phase" key
    """
    filtered_actions = []
    
    for action in actions:
        risk = action.get("risk_score", 0)
        
        # P0: Map negative space - remove high-risk actions
        if risk >= 8:
            continue  # Skip entirely (negative space)
        
        # Create copy to avoid modifying original
        action_copy = action.copy()
        
        # P1: Find safe channels
        if 4 <= risk <= 7:
            action_copy["protocol_phase"] = "channel"
            filtered_actions.append(action_copy)
        
        # P2: Optimize within channels
        elif risk <= 3:
            action_copy["protocol_phase"] = "optimize"
            filtered_actions.append(action_copy)
    
    return filtered_actions
```

### **Groq Llama-70b output:**
```python
def apply_deadband_protocol(actions):
    """
    Implements the Deadband Protocol for action filtering.
    P0: risk_score >= 8 -> removed
    P1: risk_score 4-7 -> kept, marked "channel"
    P2: risk_score 0-3 -> kept, marked "optimize"
    """
    result = []
    
    for act in actions:
        risk = act["risk_score"]
        
        # P0 elimination
        if risk >= 8:
            continue
        
        new_act = dict(act)
        if 4 <= risk <= 7:
            new_act["protocol_phase"] = "channel"
        elif 0 <= risk <= 3:
            new_act["protocol_phase"] = "optimize"
        else:
            # Handle unexpected risk scores
            new_act["protocol_phase"] = "unknown"
        
        result.append(new_act)
    
    return result
```

**Comparison:**
- **Correctness:** Both implement the logic correctly. DeepSeek uses `.get()` with default, which is more defensive. Groq assumes key exists.
- **Readability:** DeepSeek includes fleet doctrine comments ("Map negative space"). Groq is more concise.
- **Fleet context:** DeepSeek shows better awareness by using fleet terminology in comments.
- **Edge cases:** Groq adds handling for "unexpected risk scores" which could be useful but isn't specified in requirements.

## **3. Function 2: Tile Validator**

**Prompt to both models:**
```
Write a Python function `validate_tile` that checks if a training tile meets fleet submission criteria. Input: dict with keys "content" (str), "agent_id" (str), "cycle" (int), "tile_type" (str). Rules: content must be 50-5000 chars, agent_id must start with "zeroclaw-", cycle must be positive, tile_type must be in ["code", "analysis", "documentation", "test"]. Return tuple (bool, str) where bool indicates validity and str provides error message if invalid.
```

### **DeepSeek-chat output:**
```python
def validate_tile(tile_data):
    """
    Validate a training tile against fleet submission criteria.
    
    Args:
        tile_data: Dict with keys "content", "agent_id", "cycle", "tile_type"
    
    Returns:
        (is_valid, error_message)
    """
    # Check required keys
    required_keys = ["content", "agent_id", "cycle", "tile_type"]
    for key in required_keys:
        if key not in tile_data:
            return False, f"Missing required key: {key}"
    
    content = tile_data["content"]
    agent_id = tile_data["agent_id"]
    cycle = tile_data["cycle"]
    tile_type = tile_data["tile_type"]
    
    # Content length check
    if len(content) < 50:
        return False, f"Content too short ({len(content)} chars). Minimum: 50"
    if len(content) > 5000:
        return False, f"Content too long ({len(content)} chars). Maximum: 5000"
    
    # Agent ID format
    if not agent_id.startswith("zeroclaw-"):
        return False, f"Agent ID must start with 'zeroclaw-'. Got: {agent_id}"
    
    # Cycle validation
    if not isinstance(cycle, int):
        return False, f"Cycle must be integer. Got: {type(cycle)}"
    if cycle <= 0:
        return False, f"Cycle must be positive. Got: {cycle}"
    
    # Tile type validation
    valid_types = ["code", "analysis", "documentation", "test"]
    if tile_type not in valid_types:
        return False, f"Invalid tile_type: {tile_type}. Must be one of {valid_types}"
    
    return True, "Tile is valid"
```

### **Groq Llama-70b output:**
```python
def validate_tile(tile):
    """
    Validates a training tile for fleet submission.
    """
    errors = []
    
    # Content validation
    if "content" not in tile:
        errors.append("Missing 'content'")
    else:
        if len(tile["content"]) < 50:
            errors.append("Content too short (< 50 chars)")
        if len(tile["content"]) > 5000:
            errors.append("Content too long (> 5000 chars)")
    
    # Agent ID validation
    if "agent_id" not in tile:
        errors.append("Missing 'agent_id'")
    elif not tile["agent_id"].startswith("zeroclaw-"):
        errors.append("agent_id must start with 'zeroclaw-'")
    
    # Cycle validation
    if "cycle" not in tile:
        errors.append("Missing 'cycle'")
    elif not isinstance(tile["cycle"], int):
        errors.append("cycle must be an integer")
    elif tile["cycle"] <= 0:
        errors.append("cycle must be positive")
    
    # Tile type validation
    valid_types = {"code", "analysis", "documentation", "test"}
    if "tile_type" not in tile:
        errors.append("Missing 'tile_type'")
    elif tile["tile_type"] not in valid_types:
        errors.append(f"tile_type must be one of {list(valid_types)}")
    
    if errors:
        return False, "; ".join(errors)
    return True, "Valid"
```

**Comparison:**
- **Correctness:** Both are correct. DeepSeek returns on first error, Groq collects all errors.
- **Readability:** DeepSeek has more detailed docstring and comments. Groq uses concise error collection.
- **Fleet context:** Neither explicitly references fleet, but both implement the rules.
- **Edge cases:** Groq's approach of collecting all errors is more user-friendly for debugging. DeepSeek's early exit is more efficient.
