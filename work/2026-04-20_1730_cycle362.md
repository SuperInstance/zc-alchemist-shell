```python
def deadband_filter(data, lower, upper):
    """
    Apply a deadband filter to input data.
    
    If data is within [lower, upper], return the nearest bound.
    Otherwise, return data unchanged.
    
    Parameters:
    data (float): Input value
    lower (float): Lower bound of deadband
    upper (float): Upper bound of deadband
    
    Returns:
    float: Filtered value
    
    Examples:
    >>> deadband_filter(5.0, 2.0, 8.0)
    5.0
    >>> deadband_filter(3.0, 4.0, 6.0)
    4.0
    >>> deadband_filter(7.0, 4.0, 6.0)
    6.0
    """
    if lower > upper:
        lower, upper = upper, lower  # Ensure proper bounds
    
    if data < lower:
        return data
    elif data > upper:
        return data
    else:
        # Within deadband, snap to nearest bound
        if abs(data - lower) <= abs(data - upper):
            return lower
        else:
            return upper
```

```python
def deadband_filter(data, lower, upper):
    """
    Implements a deadband filter.
    
    Args:
        data: Input value (float).
        lower: Lower deadband threshold (float).
        upper: Upper deadband threshold (float).
        
    Returns:
        Filtered value (float).
        
    Example:
        >>> deadband_filter(5.0, 2.0, 8.0)
        5.0
        >>> deadband_filter(3.0, 4.0, 6.0)
        4.0
        >>> deadband_filter(7.0, 4.0, 6.0)
        6.0
    """
    # Ensure lower <= upper
    if lower > upper:
        lower, upper = upper, lower
        
    if data < lower:
        return data
    elif data > upper:
        return data
    else:
        # Within deadband: snap to nearest boundary
        dist_to_lower = abs(data - lower)
        dist_to_upper = abs(data - upper)
        return lower if dist_to_lower <= dist_to_upper else upper
```

**Comparison:**
- Both correct and nearly identical.
- DeepSeek includes explicit comment `# Within deadband, snap to nearest bound`. Groq uses variable assignments for distances.
- Both handle lower > upper swap.
- DeepSeek's docstring has more detailed parameter descriptions.
- **Verdict:** Tie on correctness. DeepSeek slightly more verbose comments.

```python
import re

def parse_fleet_log_line(line):
    """
    Parse a structured fleet log line.
    
    Expected format: [TIMESTAMP] SERVICE: LEVEL - message
    
    Parameters:
    line (str): Log line string
    
    Returns:
    dict: Parsed components with keys: timestamp, service, level, message
    
    Examples:
    >>> parse_fleet_log_line("[2026-04-20T17:30:45Z] oracle1: INFO - Service started")
    {'timestamp': '2026-04-20T17:30:45Z', 'service': 'oracle1', 'level': 'INFO', 'message': 'Service started'}
    >>> parse_fleet_log_line("[2026-04-20T17:31:00Z] jetsonclaw1: ERROR - GPU timeout")
    {'timestamp': '2026-04-20T17:31:00Z', 'service': 'jetsonclaw1', 'level': 'ERROR', 'message': 'GPU timeout'}
    """
    pattern = r'\[([^\]]+)\]\s*([^:]+):\s*([^\s-]+)\s*-\s*(.+)'
    match = re.match(pattern, line.strip())
    
    if not match:
        return {
            'timestamp': None,
            'service': None,
            'level': None,
            'message': line.strip()
        }
    
    timestamp, service, level, message = match.groups()
    return {
        'timestamp': timestamp,
        'service': service.strip(),
        'level': level.strip(),
        'message': message.strip()
    }
```

#### **Groq Llama-70b output:**
```python
import re

def parse_fleet_log_line(line):
    """
    Parses a fleet log line in format: [TIMESTAMP] SERVICE: LEVEL - message
    
    Args:
        line (str): The log line.
        
    Returns:
        dict: Dictionary with keys: timestamp, service, level, message.
        
    Example:
        >>> parse_fleet_log_line("[2026-04-20 12:34:56] oracle1: INFO - System ok")
        {'timestamp': '2026-04-20 12:34:56', 'service': 'oracle1', 'level': 'INFO', 'message': 'System ok'}
    """
    # Regex to capture components
    pattern = r'\[([^\]]+)\]\s*([^:]+):\s*([A-Za-z]+)\s*-\s*(.+)'
    match = re.match(pattern, line.strip())
    
    if match:
        timestamp, service, level, message = match.groups()
        return {
            'timestamp': timestamp.strip(),
            'service': service.strip(),
            'level': level.strip(),
            'message': message.strip()
        }
    else:
        # Return None values if parsing fails
        return {'timestamp': None, 'service': None, 'level': None, 'message': line.strip()}
```

**Comparison:**
- DeepSeek's regex: `([^\s-]+)` for level (allows non-space, non-hyphen). Groq: `([A-Za-z]+)` (letters only).
- DeepSeek's pattern more flexible for level like "WARN-2".
- Both handle failure with None dict.
- DeepSeek's example uses ISO timestamp with 'T' and 'Z'; Groq uses simpler timestamp.
- **Verdict:** DeepSeek's regex is more robust for varied level formats. Groq's `[A-Za-z]+` may break on "ERROR-404".
