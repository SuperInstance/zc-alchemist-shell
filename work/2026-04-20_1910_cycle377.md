## **2. Function 1: Deadband Protocol Validator**

**Prompt:**
```
Write a Python function `validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]`.
The function checks if a list of agent action strings follows the Cocapn fleet's Deadband Protocol.
Protocol Rules:
- P0 (map negative space) actions must start with "P0:".
- P1 (find safe channels) actions must start with "P1:".
- P2 (optimize within channels) actions must start with "P2:".
- The sequence must be non-decreasing: All P0 steps must come before any P1, and all P1 before any P2. Skipping a phase is allowed (e.g., only P1 and P2), but order must be maintained.
Return (True, "") if valid. Return (False, "error description") if invalid.
Include a comprehensive docstring and 2-3 inline comments.
```

### **DeepSeek-chat Output:**
```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """
    Validates a sequence of agent actions against the Deadband Protocol priority order.

    The Deadband Protocol requires phases to appear in order P0 -> P1 -> P2.
    Steps within a phase can be interleaved, but no higher-priority phase step
    may appear after a lower-priority phase has begun.

    Args:
        steps: List of action strings, each prefixed with 'P0:', 'P1:', or 'P2:'.

    Returns:
        Tuple of (is_valid: bool, error_message: str). Error message is empty
        string if valid.
    """
    # Track the highest phase we've seen so far
    current_phase = -1  # -1: none, 0: P0, 1: P1, 2: P2
    phase_map = {"P0": 0, "P1": 1, "P2": 2}

    for i, step in enumerate(steps):
        # Check prefix format
        if not (step.startswith("P0:") or step.startswith("P1:") or step.startswith("P2:")):
            return False, f"Step {i}: '{step}' must start with P0:, P1:, or P2:"

        # Determine phase of this step
        prefix = step[:3]  # "P0:", "P1:", or "P2:"
        step_phase = phase_map[prefix[:-1]]  # Remove colon to get key

        # Phase order violation: if we go backward in phase number
        if step_phase < current_phase:
            return (
                False,
                f"Step {i}: '{step}' violates phase order. "
                f"Current phase is P{current_phase}, cannot go back to P{step_phase}."
            )

        # Update current phase if we've moved forward
        if step_phase > current_phase:
            current_phase = step_phase

    return True, ""
```

```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """
    Validate that a list of action strings adheres to the Deadband Protocol sequence.

    Parameters:
    steps (list[str]): A list of strings where each string should start with
                       "P0:", "P1:", or "P2:" representing the phase.

    Returns:
    tuple[bool, str]: A tuple where the first element is a boolean indicating
                      validity (True if valid, False otherwise) and the second
                      element is an error message (empty string if valid).
    """
    allowed_prefixes = {"P0:", "P1:", "P2:"}
    phase_order = {"P0:": 0, "P1:": 1, "P2:": 2}

    last_phase_index = -1

    for idx, step in enumerate(steps):
        # Validate prefix
        if not any(step.startswith(prefix) for prefix in allowed_prefixes):
            return False, f"Invalid prefix at index {idx}: '{step}'. Must start with P0:, P1:, or P2:."

        # Get the phase of current step
        current_prefix = step[:3]
        current_phase = phase_order[current_prefix]

        # Check order
        if current_phase < last_phase_index:
            return False, (f"Phase order violation at index {idx}. "
                          f"Found {current_prefix} after a higher phase. "
                          f"Order must be P0 -> P1 -> P2.")
        last_phase_index = current_phase

    return True, ""
```

*   **Clarity:** DeepSeek's docstring is more detailed about protocol semantics. Llama's is standard. Both have clear inline comments.
*   **Efficiency:** Equivalent O(n).
*   **Style:** Both are PEP 8 compliant. DeepSeek uses a map without colon, Llama keeps the colon in keys.
