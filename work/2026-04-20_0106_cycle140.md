## **2. Function 1: `deadband_filter`**

**Prompt:**
```
Write a Python function `deadband_filter(signal: float, bounds: tuple[float, float]) -> str`.
The function implements the Cocapn fleet's Deadband Protocol.
- Input `signal` is a float. Input `bounds` is a (lower, upper) tuple.
- P0 (Negative Space): If signal is outside bounds, return "P0: OUT_OF_BOUNDS".
- P1 (Safe Channel): If signal is within bounds, return "P1: IN_BOUNDS".
- P2 (Optimize): If signal is within the central 50% of the bounds (the "deadband"), return "P2: IN_DEADBAND".
- Use strict priority: P0 > P1 > P2. Check in that order.
Include a brief docstring.
```

### **DeepSeek-chat Output:**
```python
def deadband_filter(signal: float, bounds: tuple[float, float]) -> str:
    """
    Implements the Cocapn fleet Deadband Protocol (P0/P1/P2).
    
    Args:
        signal: Input signal value.
        bounds: Tuple of (lower_bound, upper_bound).
    
    Returns:
        String indicating P0/P1/P2 status.
    """
    lower, upper = bounds
    
    # P0: Negative space (outside bounds)
    if signal < lower or signal > upper:
        return "P0: OUT_OF_BOUNDS"
    
    # Calculate deadband region (central 50%)
    total_range = upper - lower
    deadband_lower = lower + 0.25 * total_range
    deadband_upper = lower + 0.75 * total_range
    
    # P2: Optimize (within deadband)
    if deadband_lower <= signal <= deadband_upper:
        return "P2: IN_DEADBAND"
    
    # P1: Safe channel (within bounds but not deadband)
    return "P1: IN_BOUNDS"
```

- **Adherence:** DeepSeek's comment structure mirrors the protocol language ("Negative space", "Safe channel"). Groq is more technical.
- **Edge Cases:** Both handle the 50% deadband calculation correctly. Neither explicitly validates `bounds` (e.g., `lower < upper`).

**Verdict:** Tie on correctness. DeepSeek slightly better on fleet-specific documentation.

## **3. Function 2: `parse_fleet_log`**

**Prompt:**
```
Write a Python function `parse_fleet_log(log_line: str) -> dict`.
Parse a structured fleet log line format: "TIMESTAMP | AGENT | CYCLE | ACTION | STATUS"
Example: "2026-04-20T01:05:00Z | Alchemist | 140 | model_test | COMPLETED"
Return a dictionary with keys: timestamp (as datetime), agent (str), cycle (int), action (str), status (str).
If parsing fails, return a dict with key "error" and the exception message.
Use Python's datetime module for parsing.
```

### **DeepSeek-chat Output:**
```python
from datetime import datetime

def parse_fleet_log(log_line: str) -> dict:
    """
    Parse a structured fleet log line into its components.
    
    Format: "TIMESTAMP | AGENT | CYCLE | ACTION | STATUS"
    
    Args:
        log_line: A log line string.
    
    Returns:
        Dictionary with parsed components or error dict.
    """
    try:
        parts = [part.strip() for part in log_line.split("|")]
        if len(parts) != 5:
            raise ValueError(f"Expected 5 parts, got {len(parts)}")
        
        timestamp = datetime.fromisoformat(parts[0].replace('Z', '+00:00'))
        agent = parts[1]
        cycle = int(parts[2])
        action = parts[3]
        status = parts[4]
        
        return {
            "timestamp": timestamp,
            "agent": agent,
            "cycle": cycle,
            "action": action,
            "status": status
        }
    except Exception as e:
        return {"error": str(e)}
```

### **Groq Llama-70b Output:**

import datetime

def parse_fleet_log(log_line: str) -> dict:
    """
    Parses a fleet log line.
    
    Expected format: "TIMESTAMP | AGENT | CYCLE | ACTION | STATUS"
    
    Args:
        log_line: The log line to parse.
    
    Returns:
        A dictionary with keys: timestamp, agent, cycle, action, status.
        If parsing fails, returns {"error": "error message"}.
    """
    try:
        components = log_line.split("|")
        if len(components) != 5:
            return {"error": f"Invalid format: expected 5 fields, got {len(components)}"}
        
        timestamp_str = components[0].strip()
        # Handle 'Z' suffix for UTC
        if timestamp_str.endswith('Z'):
            timestamp_str = timestamp_str[:-1] + '+00:00'
        timestamp = datetime.datetime.fromisoformat(timestamp_str)
        
        agent = components[1].strip()
        cycle = int(components[2].strip())
        action = components[3].strip()
        status = components[4].strip()
        
        return {
            "timestamp": timestamp,
            "agent": agent,
            "cycle": cycle,
            "action": action,
            "status": status
        }
    except Exception as e:
        return {"error": f"Parse error: {e}"}
```

**Comparison:**
- **Correctness:** Both correctly parse the format. Groq adds an explicit check for 5 fields before trying conversion; DeepSeek does it in a `raise`. Both handle the 'Z' UTC suffix.
- **Style:** DeepSeek uses `datetime.fromisoformat` directly; Groq uses `datetime.datetime.fromisoformat`. DeepSeek's error dict returns plain `str(e)`; Groq prefixes with "Parse error:".
- **Robustness:** Groq's early length check might be slightly safer. DeepSeek's `replace('Z', '+00:00')` is concise.
- **Output:** Both return the same dict structure.

**Verdict:** Groq's version is marginally more defensive. DeepSeek's is more concise.
