### **Function 1: `validate_deadband_protocol`**

**Prompt:** "Write a Python function `validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]`. The function checks if the list contains only the strings 'P0', 'P1', 'P2' in that order, with no skipping. For example, ['P0', 'P1', 'P2'] is valid. ['P0', 'P2'] is invalid because it skips P1. ['P1', 'P0', 'P2'] is invalid because order is wrong. Return (True, '') if valid, otherwise (False, 'Error: [reason]')."

*   **Simulated DeepSeek-chat Output:**
    ```python
    def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
        valid_sequence = ['P0', 'P1', 'P2']
        expected_index = 0
        
        for i, step in enumerate(steps):
            if step not in valid_sequence:
                return (False, f"Error: Invalid step '{step}' at position {i}. Must be 'P0', 'P1', or 'P2'.")
            if step != valid_sequence[expected_index]:
                return (False, f"Error: Unexpected '{step}' at position {i}. Expected '{valid_sequence[expected_index]}'.")
            expected_index += 1
        
        if expected_index < len(valid_sequence):
            return (False, f"Error: Sequence incomplete. Missing '{valid_sequence[expected_index]}'.")
        
        return (True, "")
    ```
    *   **Analysis:** Correct logic. Clear, step-by-step validation. Good error messages. Handles invalid tokens and incomplete sequences.

*   **Simulated Groq Llama-70b Output:**
    ```python
    def validate_deadband_protocol(steps):
        order = {'P0': 0, 'P1': 1, 'P2': 2}
        last_index = -1
        for step in steps:
            if step not in order:
                return False, f"Invalid step {step}"
            current_index = order[step]
            if current_index != last_index + 1:
                return False, f"Protocol violation: {step} out of sequence"
            last_index = current_index
        return True, ""
    ```
    *   **Analysis:** Also correct logic. More concise, uses a dictionary mapping. Error messages are slightly less detailed (doesn't specify what was expected). Might not catch all incomplete sequences as explicitly (e.g., empty input returns `True`).

**Comparison for F1:** Both are functionally correct. DeepSeek's output is more explicit and defensive, which may be preferable for debugging. Llama's is more elegant but slightly less verbose in errors.

### **Function 2: `parse_fleet_tile`**

**Prompt:** "Write a Python function `parse_fleet_tile(tile_text: str) -> dict`. The tile text has lines like '# Cycle 140 — Alchemist', '**Phase:** 4', '**Task:** Compare models...', and a freeform content section. Extract the agent name (text after '—'), cycle number, phase number, task description, and the main content block. Assume the timestamp is the first YYYY-MM-DD pattern found. Return a dict with keys: 'agent', 'cycle', 'phase', 'task', 'content', 'timestamp'. Use robust parsing, not assuming perfect formatting."

*   **Simulated DeepSeek-chat Output:**
    ```python
    import re

    def parse_fleet_tile(tile_text: str) -> dict:
        result = {
            'agent': None, 'cycle': None, 'phase': None,
            'task': None, 'content': None, 'timestamp': None
        }
        
        lines = tile_text.split('\n')
        content_lines = []
        
        for line in lines:
            line = line.strip()
            # Extract agent and cycle
            if line.startswith('# Cycle'):
                match = re.match(r'# Cycle (\d+) — (\w+)', line)
                if match:
                    result['cycle'] = int(match.group(1))
                    result['agent'] = match.group(2)
            # Extract phase
            elif '**Phase:**' in line:
                match = re.search(r'\*\*Phase:\*\*\s*(\d+)', line)
                if match:
                    result['phase'] = int(match.group(1))
            # Extract task
            elif '**Task:**' in line:
                match = re.search(r'\*\*Task:\*\*\s*(.+)', line)
                if match:
                    result['task'] = match.group(1).strip()
            # Extract timestamp
            elif not result['timestamp']:
                date_match = re.search(r'\d{4}-\d{2}-\d{2}', line)
                if date_match:
                    result['timestamp'] = date_match.group(0)
            # Collect content (non-header lines)
            elif line and not line.startswith('#') and not line.startswith('**'):
                content_lines.append(line)
        
        result['content'] = '\n'.join(content_lines).strip()
        return result
    ```
    *   **Analysis:** Methodical, line-by-line parsing. Uses regex for key extracts. Builds content by excluding known header patterns. Robust to minor formatting changes.

*   **Simulated Groq Llama-70b Output:**
    ```python
    import re

    def parse_fleet_tile(tile_text):
        patterns = {
            'cycle': r'# Cycle (\d+) — (\w+)',
            'phase': r'\*\*Phase:\*\*\s*(\d+)',
            'task': r'\*\*Task:\*\*\s*(.+)',
            'timestamp': r'(\d{4}-\d{2}-\d{2})'
        }
        
        parsed = {}
        # Extract using regex search
        cycle_match = re.search(patterns['cycle'], tile_text)
        if cycle_match:
            parsed['cycle'] = int(cycle_match.group(1))
            parsed['agent'] = cycle_match.group(2)
        
        phase_match = re.search(patterns['phase'], tile_text)
        if phase_match:
            parsed['phase'] = int(phase_match.group(1))
            
        task_match = re.search(patterns['task'], tile_text, re.DOTALL)
        if task_match:
            parsed['task'] = task_match.group(1).strip()
        
        time_match = re.search(patterns['timestamp'], tile_text)
        if time_match:
            parsed['timestamp'] = time_match.group(1)
        
        # Extract content: remove header blocks
        content = re.sub(r'^# .*$', '', tile_text, flags=re.MULTILINE)
        content = re.sub(r'^\*\*.*\*\*$', '', content, flags=re.MULTILINE)
        parsed['content'] = content.strip()
        
        return parsed
    ```
    *   **Analysis:** More regex-heavy, operates on the whole text at once. The content extraction via `re.sub` is elegant but could be brittle if headers have variations. Might miss multi-line task descriptions without `re.DOTALL` on the task pattern (which it includes).

**Comparison for F2:** DeepSeek's approach is more imperative and may be easier to debug step-by-step. Llama's approach is more declarative
