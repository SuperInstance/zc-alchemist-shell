## **2. Function 1: Deadband Protocol Validator**

**Prompt:** "Write a Python function `validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]`. The Deadband Protocol has three strict priority phases: P0 (map negative space), P1 (find safe channels), P2 (optimize within channels). A valid sequence must start with P0, may have multiple P0 steps, then may transition to P1, then to P2. It cannot go backwards (e.g., P1 to P0) or skip phases (e.g., P0 to P2). Return True and an empty string if valid. Return False and a diagnostic message (e.g., 'Sequence starts with P1', 'Illegal transition from P1 to P0') if invalid."

**DeepSeek-chat Implementation:**
```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """
    Validates a sequence of Deadband Protocol steps.
    Rules: Must start with P0. Transitions allowed: P0->P0, P0->P1, P1->P1, P1->P2, P2->P2.
    Returns (is_valid, diagnostic_message).
    """
    if not steps:
        return False, "Empty sequence"

    allowed_transitions = {
        'P0': {'P0', 'P1'},
        'P1': {'P1', 'P2'},
        'P2': {'P2'}
    }

    current_phase = None
    for i, step in enumerate(steps):
        step = step.strip().upper()
        if step not in ('P0', 'P1', 'P2'):
            return False, f"Invalid step '{step}' at index {i}"

        if i == 0:
            if step != 'P0':
                return False, f"Sequence must start with P0, got '{step}'"
            current_phase = step
            continue

        if step not in allowed_transitions[current_phase]:
            return False, f"Illegal transition from {current_phase} to {step} at index {i}"
        current_phase = step

    return True, ""
```

```python
def validate_deadband_protocol(steps):
    """
    Check if the steps follow Deadband Protocol rules.
    P0 -> P1 -> P2, no skipping, no going back.
    """
    if len(steps) == 0:
        return False, "no steps provided"

    # Normalize steps
    normalized = [s.strip().upper() for s in steps]
    valid_steps = {"P0", "P1", "P2"}
    for s in normalized:
        if s not in valid_steps:
            return False, f"invalid step {s}"

    # Check order
    phase_order = {"P0": 0, "P1": 1, "P2": 2}
    for i in range(len(normalized) - 1):
        curr = normalized[i]
        nxt = normalized[i + 1]
        if phase_order[nxt] < phase_order[curr]:
            return False, f"cannot go back from {curr} to {nxt}"
        if phase_order[nxt] > phase_order[curr] + 1:
            return False, f"cannot skip from {curr} to {nxt}"

    # First step must be P0
    if normalized[0] != "P0":
        return False, "must start with P0"

    return True, ""
```

*   **Correctness:** Both are logically correct and enforce the rules.
*   **Style:** DeepSeek uses an explicit transition map (`allowed_transitions`) which is more extensible if phases change. Groq uses numeric ordering (`phase_order`) which is concise for this specific linear protocol.
*   **Robustness:** DeepSeek checks for invalid step strings earlier in the loop. Groq does a separate validation pass first.
*   **Output:** Both provide clear diagnostic messages. DeepSeek's messages include index, which might be more helpful for debugging.
*   **Verdict:** DeepSeek's version is slightly more defensive and extensible. Groq's is clean and mathematically straightforward. Tie on correctness; edge to DeepSeek for maintainability.

## **3. Function 2: Fleet Tile Parser**

**Prompt:** "Write a Python function `parse_fleet_tile(tile_text: str) -> dict`. A training tile text has headers like '# Cycle 356 — Alchemist', '**Phase:** 4', '**Task:** ...', and a content section. Parse it to return a dictionary with keys: 'agent', 'cycle', 'phase', 'task', 'content'. Assume the format is consistent but may have extra blank lines. The content is everything after the task line until the end. Use robust string splitting and regex if needed."

**DeepSeek-chat Implementation:**
```python
import re

def parse_fleet_tile(tile_text: str) -> dict:
    """
    Parses a fleet training tile text into structured metadata.
    Expected format includes lines like:
    # Cycle N — AgentName
    **Phase:** X
    **Task:** Task description
    ...content...
    """
    lines = tile_text.strip().split('\n')
    result = {
        'agent': 'Unknown',
        'cycle': -1,
        'phase': -1,
        'task': '',
        'content': ''
    }

    for i, line in enumerate(lines):
        line = line.strip()
        # Parse agent and cycle from header
        if line.startswith('# Cycle'):
            match = re.match(r'# Cycle (\d+) — (\w+)', line)
            if match:
                result['cycle'] = int(match.group(1))
                result['agent'] = match.group(2)
        # Parse phase
        elif line.startswith('**Phase:**'):
            match = re.search(r'**Phase:**\s*(\d+)', line)
            if match:
                result['phase'] = int(match.group(1))
        # Parse task
        elif line.startswith('**Task:**'):
            result['task'] = line.replace('**Task:**', '').strip()
            # Content starts after this line
            content_lines = []
            for j in range(i + 1, len(lines)):
                content_lines.append(lines[j])
            result['content'] = '\n'.join(content_lines).strip()
            break  # Stop parsing after task line

    return result
```
