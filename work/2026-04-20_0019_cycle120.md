## **2. Function 1: `deadband_filter`**

**Prompt:** "Write a Python function called `deadband_filter` that implements the Cocapn fleet's Deadband Protocol. The function takes a list of floating-point numbers `decisions` and returns a filtered list. The protocol has three strict priorities: P0 (negative space): remove any values where `abs(value) < 0.1`. P1 (safe channels): from the remaining values, keep only those where `0.1 <= abs(value) <= 1.0`. P2 (optimize): return the list sorted by absolute value. Include a descriptive docstring and example usage."

### **DeepSeek-chat Implementation:**
```python
def deadband_filter(decisions):
    """
    Apply the Cocapn fleet's Deadband Protocol to a list of decision values.

    Protocol:
        P0 (Negative Space): Remove values within the deadband (|value| < 0.1).
        P1 (Safe Channels): Keep only values within the operational channel (0.1 <= |value| <= 1.0).
        P2 (Optimize): Sort remaining values by their absolute magnitude.

    Args:
        decisions (list[float]): Input list of decision values.

    Returns:
        list[float]: Filtered and sorted list according to the Deadband Protocol.

    Example:
        >>> deadband_filter([-0.05, 0.0, 0.15, -0.8, 1.5, 0.3])
        [0.15, 0.3, -0.8]
    """
    # P0: Map negative space - remove values in deadband
    filtered = [v for v in decisions if abs(v) >= 0.1]
    
    # P1: Find safe channels - keep only values in operational range
    filtered = [v for v in filtered if abs(v) <= 1.0]
    
    # P2: Optimize within channels - sort by absolute value
    filtered.sort(key=abs)
    
    return filtered
```

## **3. Function 2: `parse_fleet_log`**

**Prompt:** "Write a Python function `parse_fleet_log` that parses a single line of Cocapn fleet log data. The log format is: `'YYYY-MM-DD HH:MM:SS | AGENT_NAME | ACTION | DETAILS'`. The function should return a dictionary with keys: `timestamp` (datetime object), `agent` (string), `action` (string), `details` (string). If the line doesn't match the expected format, return `None`. Use the `datetime` module. Include error handling."

### **DeepSeek-chat Implementation:**
```python
from datetime import datetime

def parse_fleet_log(log_line):
    """
    Parse a single line of Cocapn fleet log data.

    Expected format: 'YYYY-MM-DD HH:MM:SS | AGENT_NAME | ACTION | DETAILS'

    Args:
        log_line (str): A line of fleet log data.

    Returns:
        dict or None: Dictionary with keys 'timestamp' (datetime), 'agent' (str),
                     'action' (str), 'details' (str). Returns None if parsing fails.
    """
    # Basic format check: must have three separators
    if log_line.count(' | ') != 3:
        return None
    
    try:
        parts = log_line.split(' | ', maxsplit=3)
        date_str, agent, action, details = parts
        
        # Parse timestamp
        timestamp = datetime.strptime(date_str, '%Y-%m-%d %H:%M:%S')
        
        return {
            'timestamp': timestamp,
            'agent': agent.strip(),
            'action': action.strip(),
            'details': details.strip()
        }
    except (ValueError, IndexError):
        # Catch datetime parsing errors or split issues
        return None
```

```python
import datetime

def parse_fleet_log(line):
    """
    Parses a fleet log line into its components.

    Format: "2026-04-20 12:34:56 | Oracle1 | boot | System initialized"
    
    Args:
        line (str): The log line to parse.
    
    Returns:
        dict: Contains timestamp (datetime), agent (str), action (str), details (str).
              Returns None if format is invalid.
    """
    if not line or ' | ' not in line:
        return None
    
    segments = line.split(' | ', 3)
    if len(segments) != 4:
        return None
    
    date_part, agent_part, action_part, details_part = segments
    
    try:
        dt = datetime.datetime.strptime(date_part.strip(), '%Y-%m-%d %H:%M:%S')
    except ValueError:
        return None
    
    return {
        'timestamp': dt,
        'agent': agent_part.strip(),
        'action': action_part.strip(),
        'details': details_part.strip()
    }
```

*   **Error Handling:** DeepSeek uses a single `try-except` for `ValueError` and `IndexError`. Groq has more granular checks (`if not line`, length check) before the `try`. Both robust.
*   **Verdict:** Very similar in quality. DeepSeek's approach is slightly more concise; Groq's is slightly more defensive. Both would work in production.
