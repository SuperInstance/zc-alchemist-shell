### **Function 1: `deadband_filter`**

**Prompt:**
```
Write a Python function called `deadband_filter` that implements the Cocapn fleet's Deadband Protocol.

Input: A list of floats `decisions` representing potential actions or scores, and three float thresholds: `p0_threshold`, `p1_min`, `p1_max`.

The Deadband Protocol has three strict priority levels:
P0 (Negative Space): Any decision with absolute value <= `p0_threshold` is set to 0.0 (avoid/neutral).
P1 (Safe Channel): For decisions that survive P0, if the value is between `p1_min` and `p1_max` (inclusive), it passes through unchanged. Otherwise, it is clipped to the nearest bound (`p1_min` or `p1_max`).
P2 (Optimize): Not implemented here; P2 would happen later within the safe channel.

The function must process the list in order (P0 then P1) and return a new list of filtered/clipped values.

Include a clear docstring and a simple example in the docstring.
```

**DeepSeek-chat Output:**
```python
def deadband_filter(decisions, p0_threshold, p1_min, p1_max):
    """
    Apply the Cocapn fleet Deadband Protocol to a list of decision scores.

    Parameters:
    decisions (list of float): Input decision scores.
    p0_threshold (float): P0 threshold. Values with abs(value) <= p0_threshold are set to 0.0.
    p1_min (float): Lower bound of the P1 safe channel.
    p1_max (float): Upper bound of the P1 safe channel.

    Returns:
    list of float: Filtered decisions after applying P0 and P1 rules.

    Example:
    >>> deadband_filter([-0.1, 0.05, 1.5, 2.5, -2.0], 0.1, 0.5, 2.0)
    [0.0, 0.0, 1.5, 2.0, -2.0]
    """
    filtered = []
    for d in decisions:
        # P0: Map negative space
        if abs(d) <= p0_threshold:
            filtered.append(0.0)
            continue
        # P1: Find safe channel
        if d < p1_min:
            filtered.append(p1_min)
        elif d > p1_max:
            filtered.append(p1_max)
        else:
            filtered.append(d)
    return filtered
```

**Groq Llama-3.3-70b-Specdec Output:**

```python
def deadband_filter(decisions, p0_threshold, p1_min, p1_max):
    """
    Apply the Deadband Protocol (P0 and P1) to a list of decisions.

    Args:
        decisions: List of float decision values.
        p0_threshold: Threshold for P0 (negative space). Decisions with |value| <= p0_threshold become 0.
        p1_min: Minimum safe channel bound for P1.
        p1_max: Maximum safe channel bound for P1.

    Returns:
        List of filtered decisions.

    Example:
        >>> deadband_filter([-0.2, 0.05, 1.2, 3.0], 0.1, 0.5, 2.0)
        [-0.5, 0.0, 1.2, 2.0]
    """
    result = []
    for d in decisions:
        # P0: eliminate negative space
        if abs(d) <= p0_threshold:
            result.append(0.0)
            continue

        # P1: constrain to safe channel
        if d < p1_min:
            d = p1_min
        elif d > p1_max:
            d = p1_max
        result.append(d)
    return result
```

**Analysis for Function 1:**
*   **Correctness:** Both are logically correct and implement the protocol. DeepSeek's example is more detailed.
*   **Style:** DeepSeek uses `filtered` variable, Groq uses `result`. Both are fine.
*   **Edge Cases:** Both handle the P0 `continue` correctly. Groq's example contains an error: `-0.2` with `p0_threshold=0.1` should become `0.0`, not `-0.5`. This suggests the model might not have fully validated its own example.
*   **Adherence:** DeepSeek's docstring explicitly mentions "Cocapn fleet" and the three priority levels, showing slightly better context recall.

### **Function 2: `parse_fleet_log`**

**Prompt:**
```
Write a Python function called `parse_fleet_log` that parses a structured log line from the Cocapn fleet.

Log format (string): "TIMESTAMP | AGENT_NAME | CYCLE | STATUS | MESSAGE"
Example: "2026-04-20T01:16:00Z | Alchemist | 142 | ACTIVE | Submitted tile #3"

The function should take a log string, split it by " | " delimiter, and return a dictionary with keys:
timestamp (string), agent (string), cycle (int), status (string), message (string).

If the line does not have exactly 5 parts, raise a ValueError with a helpful message.

Include type hints and a simple test in the docstring.
```

**DeepSeek-chat Output:**
```python
from typing import Dict

def parse_fleet_log(log_line: str) -> Dict[str, str | int]:
    """
    Parse a structured Cocapn fleet log line.

    Parameters:
    log_line (str): Log line in format "TIMESTAMP | AGENT_NAME | CYCLE | STATUS | MESSAGE"

    Returns:
    dict: Parsed log with keys 'timestamp', 'agent', 'cycle', 'status', 'message'.

    Raises:
    ValueError: If log line does not contain exactly 5 parts.

    Example:
    >>> parse_fleet_log("2026-04-20T01:16:00Z | Alchemist | 142 | ACTIVE | Submitted tile #3")
    {'timestamp': '2026-04-20T01:16:00Z', 'agent': 'Alchemist', 'cycle': 142, 'status': 'ACTIVE', 'message': 'Submitted tile #3'}
    """
    parts = log_line.split(" | ")
    if len(parts) != 5:
        raise ValueError(f"Log line must have exactly 5 parts separated by ' | '. Got {len(parts)} parts: {log_line}")
    timestamp, agent, cycle_str, status, message = parts
    return {
        "timestamp": timestamp,
        "agent": agent,
        "cycle": int(cycle_str),
        "status": status,
        "message": message
    }
```
