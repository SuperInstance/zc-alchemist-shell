## **2. Function 1: `deadband_filter`**

**Prompt:**  
```
Write a Python function `deadband_filter` that implements the Cocapn fleet's Deadband Protocol.

Input: `values` (list of floats), `deadband` (float, default 0.1)
Process:
- P0 (negative space): Remove any values where `abs(value) < deadband` (too close to zero, unstable).
- P1 (safe channels): Keep only values where `deadband <= abs(value) <= 1.0` (within safe operating range).
- P2 (optimize): From remaining values, return the one closest to 0.5 (optimal center).

Return: The single optimal float, or `None` if no values survive P0/P1.

Include a brief docstring and example usage.
```

### **DeepSeek-chat output:**
```python
def deadband_filter(values, deadband=0.1):
    """
    Apply Deadband Protocol (P0-P2) to select optimal value.
    
    P0: Remove values in deadband zone (|value| < deadband).
    P1: Keep only values in safe channel (deadband <= |value| <= 1.0).
    P2: Choose value closest to 0.5 (optimal center).
    
    Args:
        values: List of float values.
        deadband: Deadband threshold (default 0.1).
    
    Returns:
        Optimal float or None if no values survive.
    """
    # P0: Map negative space
    p0_safe = [v for v in values if abs(v) >= deadband]
    if not p0_safe:
        return None
    
    # P1: Find safe channels
    p1_safe = [v for v in p0_safe if abs(v) <= 1.0]
    if not p1_safe:
        return None
    
    # P2: Optimize within channels
    optimal = min(p1_safe, key=lambda x: abs(x - 0.5))
    return optimal

# Example
if __name__ == "__main__":
    test_vals = [0.05, 0.2, 0.8, 1.2, -0.3]
    result = deadband_filter(test_vals)
    print(f"Optimal value: {result}")  # Expected: 0.2 (closest to 0.5 within safe range)
```

### **Groq Llama-70b output:**
```python
def deadband_filter(values, deadband=0.1):
    """
    Implements the Deadband Protocol (P0, P1, P2) for value selection.
    
    Parameters:
    - values: list of floats
    - deadband: float, default 0.1
    
    Returns:
    - float or None
    """
    # P0: eliminate values in the deadband (too close to zero)
    filtered = [v for v in values if abs(v) >= deadband]
    
    # P1: only allow values within the safe operating range [-1.0, 1.0]
    filtered = [v for v in filtered if abs(v) <= 1.0]
    
    if not filtered:
        return None
    
    # P2: select the value closest to the ideal center (0.5)
    best = min(filtered, key=lambda x: abs(x - 0.5))
    return best

# Example usage
sample = [0.05, 0.2, 0.8, 1.2, -0.3]
print(deadband_filter(sample))  # 0.2
```

**Comparison:**
- Both are logically correct and produce the same result.
- DeepSeek includes a more detailed docstring with P0/P1/P2 explanation.
- DeepSeek adds a `if __name__ == "__main__"` guard for example execution.
- Groq's variable naming (`filtered` reused) is slightly less clear than DeepSeek's (`p0_safe`, `p1_safe`).
- Both follow PEP8.

## **3. Function 2: `parse_fleet_log`**

**Prompt:**  
```
Write a Python function `parse_fleet_log` that parses a structured fleet log line.

Log format: "TIMESTAMP | AGENT | CYCLE | ACTION | DETAILS"
Example: "2026-04-19T23:45:00Z | Alchemist | 96 | model_test | deepseek vs groq"

The function should:
- Split by " | " delimiter.
- Return a dict with keys: timestamp (as string), agent, cycle (int), action, details.
- Handle malformed lines (e.g., missing parts) by returning None.
- Strip whitespace from each field.

Include error handling and a test.
```

### **DeepSeek-chat output:**
```python
def parse_fleet_log(log_line):
    """
    Parse a structured fleet log line into a dictionary.
    
    Expected format: "TIMESTAMP | AGENT | CYCLE | ACTION | DETAILS"
    
    Args:
        log_line (str): A single log line.
    
    Returns:
        dict: Parsed fields or None if malformed.
    """
    if not isinstance(log_line, str):
        return None
    
    parts = log_line.split(" | ")
    if len(parts) != 5:
        return None
    
    try:
        parsed = {
            "timestamp": parts[0].strip(),
            "agent": parts[1].strip(),
            "cycle": int(parts[2].strip()),
            "action": parts[3].strip(),
            "details": parts[4].strip()
        }
        return parsed
    except (ValueError, AttributeError):
        return None

# Test

    test_log = "2026-04-19T23:45:00Z | Alchemist | 96 | model_test | deepseek vs groq"

    print(result)
    # {'timestamp': '2026-04-19T23:45:00Z', 'agent': 'Alchemist', 'cycle': 96, 'action': 'model_test', 'details': 'deepseek vs groq'}
    
    # Malformed test

```

### **Groq Llama-70b output:**
```python
def parse_fleet_log(line):
    """
    Parses a fleet log line.
    
    Format: "TIMESTAMP | AGENT | CYCLE | ACTION | DETAILS"
    
    Returns a dictionary with keys: timestamp, agent, cycle, action, details.
    Returns None if the line is malformed.
    """
    if not line or " | " not in line:
        return None
    
    segments = line.split(" | ")
    if len(segments) != 5:
        return None
    
    timestamp, agent, cycle_str, action, details = segments
    
    try:
        cycle = int(cycle_str.strip())
    except ValueError:
        return None
    
    return {
        "timestamp": timestamp.strip(),
        "agent": agent.strip(),
        "cycle": cycle,
        "action": action.strip(),
        "details": details.strip()
    }

# Example
log = "2026-04-19T23:45:00Z | Alchemist | 96 | model_test | deepseek vs groq"
print(parse_fleet_log(log))
```

**Comparison:**
- Both correctly parse the format.
- DeepSeek includes explicit type checking (`isinstance(log_line, str)`).
- DeepSeek's error handling wraps the entire parsing in a try-except, catching more edge cases (e.g., `.strip()` on non-string).
- Groq's check `if not line or " | " not in line` is efficient but could miss cases where line has wrong delimiter count.
- DeepSeek provides a more complete test with malformed input.
