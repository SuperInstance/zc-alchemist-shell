## **2. Function 1: Deadband Filter**

**Prompt:**
"Write a Python function called `apply_deadband_protocol`. It takes one argument, `actions`, which is a list of dictionaries. Each dictionary has keys `'name'` (string) and `'risk'` (int, 0-2 where 0=P0 violation, 1=P1 safe, 2=P2 optimizable). The function must return a new list containing only the actions where `'risk'` is 1 or 2, **but only if** there are no actions with `'risk'` == 0 in the input. If any P0 (risk 0) action is present, the function must return an empty list `[]`. This enforces 'never skip to P2' by blocking all output if a P0 violation exists."

**DeepSeek-chat Output:**
```python
def apply_deadband_protocol(actions):
    """
    Apply the Deadband Protocol (P0/P1/P2) to a list of actions.
    
    Args:
        actions: List of dicts with 'name' (str) and 'risk' (int, 0-2)
        
    Returns:
        List of safe/optimizable actions, or empty list if any P0 exists.
    """
    # P0: Check for any violations
    for action in actions:
        if action.get('risk') == 0:
            return []  # P0 violation found, block everything
    
    # P1/P2: Filter for safe channels and optimizable paths
    filtered = [action for action in actions if action.get('risk') in (1, 2)]
    return filtered
```

**Groq Llama-70b Output:**
```python
def apply_deadband_protocol(actions):
    # First, check for any P0 (risk 0) actions
    has_p0 = any(action.get('risk', -1) == 0 for action in actions)
    if has_p0:
        return []  # If any P0, return empty list
    
    # Otherwise, filter for P1 and P2 actions
    result = []
    for action in actions:
        risk = action.get('risk', -1)
        if risk == 1 or risk == 2:
            result.append(action)
    return result
```

**Comparison:**
*   **Correctness:** Both are logically correct and implement the P0-check-first rule.
*   **Conciseness:** DeepSeek uses a list comprehension for filtering, which is slightly more Pythonic. Groq uses an explicit loop.
*   **Edge Handling:** DeepSeek uses `.get('risk')` which defaults to `None`; Groq defaults to `-1`. DeepSeek's approach is safer for missing keys, as `None in (1,2)` is `False`. Groq's `-1` would also be filtered out.
*   **Readability:** Both are clear. DeepSeek includes a docstring.

**Verdict:** DeepSeek's version is marginally better due to the docstring and list comprehension. Both are acceptable.

## **3. Function 2: Tile Validator**

**Prompt:**
"Write a Python function called `validate_tile`. It takes a string `tile_text`. It should return a tuple `(is_valid, reason)`. The tile is valid if: 1) It is not empty. 2) It contains at least one markdown header (`#`). 3) It does not contain the phrase 'TODO' or 'FIXME'. 4) It is under 5000 characters. The `reason` should be a short string explaining failure, or `'OK'` if valid."

**DeepSeek-chat Output:**
```python
def validate_tile(tile_text):
    """
    Validate a training tile against fleet submission criteria.
    
    Args:
        tile_text (str): The tile content to validate.
        
    Returns:
        tuple: (bool is_valid, str reason)
    """
    if not tile_text or not tile_text.strip():
        return False, "Tile is empty"
    
    if len(tile_text) > 5000:
        return False, "Tile exceeds 5000 characters"
    
    if '# ' not in tile_text:
        return False, "Tile lacks a markdown header (# )"
    
    forbidden = ['TODO', 'FIXME']
    for word in forbidden:
        if word in tile_text.upper():
            return False, f"Tile contains forbidden word '{word}'"
    
    return True, "OK"
```

**Groq Llama-70b Output:**
```python
def validate_tile(tile_text):
    # Check 1: Not empty
    if tile_text is None or len(tile_text.strip()) == 0:
        return (False, "Tile is empty or None")
    
    # Check 2: Has markdown header
    if '#' not in tile_text:
        return (False, "No markdown header found")
    
    # Check 3: No TODO/FIXME
    upper_tile = tile_text.upper()
    if 'TODO' in upper_tile or 'FIXME' in upper_tile:
        return (False, "Contains TODO or FIXME")
    
    # Check 4: Length
    if len(tile_text) > 5000:
        return (False, "Length exceeds 5000 characters")
    
    return (True, "OK")
```

**Comparison:**
*   **Correctness:** Both are mostly correct. **Critical Difference:** Groq's header check (`'#' not in tile_text`) is flawed. It would match a `#` inside a code block or comment, and the prompt specified a header (`# ` with a space). DeepSeek correctly uses `'# '`.
*   **Edge Handling:** DeepSeek explicitly checks for `None` via `not tile_text`. Groq does a separate `None` check. Both handle empty strings.
*   **Logic:** DeepSeek's forbidden word check is more extensible (loop over list). Groq's is hardcoded.
*   **Order:** Groq checks length last; DeepSeek checks it earlier (more efficient for long invalid tiles).

**Verdict:** DeepSeek's implementation is more accurate and robust. Groq's header check is a significant functional bug for this specification.
