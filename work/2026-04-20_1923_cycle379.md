### **Function 1: `validate_deadband_protocol`**

**DeepSeek-chat Output:**
```python
def validate_deadband_protocol(steps: list[str]) -> tuple[bool, str]:
    """
    Validates if a sequence of steps follows the Deadband Protocol.
    Protocol requires strict P0 -> P1 -> P2 order and only these three steps.
    
    Args:
        steps: List of step strings (e.g., ["P0", "P1", "P2"])
    
    Returns:
        Tuple of (is_valid, error_message)
    """
    valid_steps = {"P0", "P1", "P2"}
    expected_order = ["P0", "P1", "P2"]
    
    # Check for invalid steps
    for i, step in enumerate(steps):
        if step not in valid_steps:
            return False, f"Invalid step '{step}' at position {i}. Must be P0, P1, or P2."
    
    # Check order
    last_index = -1
    for step in steps:
        current_index = expected_order.index(step)
        if current_index < last_index:
            return False, f"Protocol violation: {step} appears out of order. Strict P0->P1->P2 required."
        last_index = current_index
    
    return True, ""
```

**Groq Llama-70b Output:**
```python
def validate_deadband_protocol(steps):
    """
    Validate Deadband Protocol steps.
    Rules:
    1. Only "P0", "P1", "P2" allowed.
    2. Must follow order P0 -> P1 -> P2 (can skip but not reverse).
    """
    allowed = {"P0", "P1", "P2"}
    order = {"P0": 0, "P1": 1, "P2": 2}
    
    if not steps:
        return True, "Empty sequence"
        
    prev_level = -1
    for idx, step in enumerate(steps):
        if step not in allowed:
            return False, f"Invalid step {step} at index {idx}"
        curr_level = order[step]
        if curr_level < prev_level:
            return False, f"Order violation at index {idx}: {step} after higher priority step"
        prev_level = curr_level
    return True, ""
```

**Comparison:**
*   **Correctness:** Both are correct for the core logic. DeepSeek's spec said "strictly follows P0→P1→P2", which its code enforces (cannot skip). Groq's comment says "can skip but not reverse", and its code allows skipping (e.g., `["P0", "P2"]` passes). This is a **spec ambiguity**. For strict fleet doctrine ("Never skip to P2"), DeepSeek's interpretation is more accurate.
*   **Clarity:** DeepSeek adds a docstring with Args/Returns. Groq's is shorter.
*   **Robustness:** DeepSeek handles empty list implicitly (loop doesn't run, returns True). Groq explicitly checks for empty list. Both fine.
*   **Idiomatic:** Both use sets for membership. DeepSeek uses `expected_order.index()` repeatedly (O(n²) worst-case). Groq uses a precomputed `order` dict (O(1) lookup). Groq's approach is more efficient.

### **Function 2: `parse_fleet_tile`**

**DeepSeek-chat Output:**
```python
import re

def parse_fleet_tile(tile_text: str) -> dict:
    """
    Parse a fleet training tile text into structured data.
    
    Expected format (flexible):
        # Cycle N — Agent
        **Phase X: ...**
        **Task:** ...
        Content...
    
    Returns dict with keys: cycle, agent, phase, task, content
    """
    lines = tile_text.strip().split('\n')
    result = {
        "cycle": None,
        "agent": None,
        "phase": None,
        "task": "",
        "content": ""
    }
    
    # Try to extract cycle and agent from first line
    first_line = lines[0] if lines else ""
    cycle_match = re.search(r'Cycle\s+(\d+)', first_line, re.IGNORECASE)
    agent_match = re.search(r'—\s*(\w+)', first_line)
    
    if cycle_match:
        result["cycle"] = int(cycle_match.group(1))
    if agent_match:
        result["agent"] = agent_match.group(1)
    
    # Look for phase and task in any line
    for line in lines:
        phase_match = re.search(r'Phase\s+(\d+)', line, re.IGNORECASE)
        if phase_match and result["phase"] is None:
            result["phase"] = int(phase_match.group(1))
        
        task_match = re.search(r'Task[:\s]*([^\n]+)', line, re.IGNORECASE)
        if task_match and not result["task"]:
            result["task"] = task_match.group(1).strip()
    
    # Content is everything else, cleaned
    content_lines = []
    for line in lines:
        if not (re.match(r'#.*Cycle|Phase|Task', line, re.IGNORECASE)):
            content_lines.append(line)
    result["content"] = '\n'.join(content_lines).strip()
    
    return result
```
