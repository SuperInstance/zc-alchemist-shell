### **Function 1: `deadband_filter`**

**Prompt (same for both models):**
```
Write a Python function `deadband_filter(data, lower, upper)` that classifies numeric values according to the Deadband Protocol:
- P0 (negative space): value < lower
- P1 (safe channel): lower <= value <= upper
- P2 (optimize): value > upper
Return a list of tuples [(value, classification_str)].
```

**DeepSeek-chat Output (A1):**
```python
def deadband_filter(data, lower, upper):
    """
    Classify numeric values into P0, P1, P2 based on deadband thresholds.
    """
    result = []
    for val in data:
        if val < lower:
            classification = "P0"
        elif lower <= val <= upper:
            classification = "P1"
        else:  # val > upper
            classification = "P2"
        result.append((val, classification))
    return result
```

**Groq Llama-70b Output (B1):**
```python

    classifications = []
```

**My Baseline (C1):**
```python
def deadband_filter(data, lower, upper):
    if lower > upper:
        raise ValueError("lower must be <= upper")
    results = []
    for v in data:
        if not isinstance(v, (int, float)):
            raise TypeError(f"Non-numeric value: {v}")
        if v < lower:
            results.append((v, "P0"))
        elif v <= upper:  # Handles lower <= v <= upper
            results.append((v, "P1"))
        else:
            results.append((v, "P2"))
    return results
```

**Comparison (A1 vs B1):**
*   **Correctness:** Both are logically correct for the core classification. My baseline adds input validation (type/order checks), which both models omitted.
*   **Conciseness:** B1 is slightly more concise (direct append). A1 uses a temporary `classification` variable, which is clearer for modification.
*   **Adherence:** Both follow the spec exactly. Neither added docstrings beyond the prompt (A1 has a basic one).
*   **Edge Cases:** Both handle the boundary `value == upper` as P1 correctly. Neither checks `lower > upper` or non-numeric data.
*   **Winner:** Tie on core logic. **DeepSeek** slightly better for readability; **Groq** slightly more concise.

### **Function 2: `parse_fleet_context`**

**Prompt:**
```
Write a Python function `parse_fleet_context(markdown_text)` that parses fleet context markdown.
Extract:
1. All section headers (##) as a list.
2. Key technologies (bulleted list under "Key Technologies") as a list.
3. Rules (bulleted list under "Rules") as a list.
Return a dict: {'sections': [...], 'technologies': [...], 'rules': [...]}.
Assume the markdown follows FLEET-CONTEXT.md format.
```

**DeepSeek-chat Output (A2):**
```python
import re

def parse_fleet_context(markdown_text):
    sections = re.findall(r'^##\s+(.+)$', markdown_text, re.MULTILINE)
    
    # Extract key technologies
    tech_section = re.search(r'## Key Technologies\n(.*?)(?=\n##|\Z)', markdown_text, re.DOTALL)
    technologies = []
    if tech_section:
        tech_items = re.findall(r'-\s*\*\*(.*?)\*\*:\s*(.*)', tech_section.group(1))
        technologies = [f"{name}: {desc}" for name, desc in tech_items]
    
    # Extract rules
    rules_section = re.search(r'## Rules\n(.*?)(?=\n##|\Z)', markdown_text, re.DOTALL)
    rules = []
    if rules_section:
        rules = re.findall(r'-\s*(.*)', rules_section.group(1))
    
    return {
        'sections': sections,
        'technologies': technologies,
        'rules': rules
    }
```

**Groq Llama-70b Output (B2):**
```python
def parse_fleet_context(markdown_text):
    lines = markdown_text.split('\n')
    sections = []
    technologies = []
    rules = []
    current_section = None
    
    for line in lines:
        if line.startswith('## '):
            sections.append(line[3:])
            current_section = line[3:]
        elif line.startswith('- **'):
            if current_section == 'Key Technologies':
                tech = line.strip('- **').split(':**')
                if len(tech) == 2:
                    technologies.append(tech[0].strip() + ': ' + tech[1].strip())
        elif line.startswith('- ') and not line.startswith('- **'):
            if current_section == 'Rules':
                rules.append(line[2:])
    
    return {'sections': sections, 'technologies': technologies, 'rules': rules}
```

**My Baseline (C2):**
```python
def parse_fleet_context(markdown_text):
    lines = markdown_text.split('\n')
    sections = []
    tech_mode = rule_mode = False
    technologies = []
    rules = []
    
    for line in lines:
        stripped = line.strip()
        # Detect section headers
        if stripped.startswith('## '):
            sec_title = stripped[3:].strip()
            sections.append(sec_title)
            tech_mode = (sec_title == "Key Technologies")
            rule_mode = (sec_title == "Rules")
            continue
        
        # Collect items
        if tech_mode and stripped.startswith('- **'):
            # Format: "- **name**: description"
            parts = stripped[2:].split(':**', 1)
            if len(parts) == 2:
                name = parts[0].strip('** ')
                desc = parts[1].strip()
                technologies.append(f"{name}: {desc}")
        elif rule_mode and stripped.startswith('- '):
            rules.append(stripped[2:])
    
    return {'sections': sections, 'technologies': technologies, 'rules': rules}
```

**Comparison (A2 vs B2):**
*   **Correctness:** A2 uses regex which may be brittle if markdown formatting varies. B2 uses line-by-line parsing with state tracking, which is more robust to minor formatting changes. Both assume specific section titles.
*   **Conciseness:** A2 is more compact using regex. B2 is longer but clearer step-by-step.
*   **Adherence:** Both extract the three requested fields. A2's regex for technologies captures `**name**: desc` pattern well. B2's logic for distinguishing technology bullets from rule bullets is correct.
*   **Edge Cases:** A2 might fail if sections are not separated by `##`. B2's `current_section` tracking could be confused by nested bullets.
*   **Winner:** **Groq Llama-70b (B2)** for more robust line-by-line parsing. DeepSeek's regex approach is clever but potentially fragile.
