## **2. Function 1: Deadband Filter**

**Spec:**  
Input: `signal: Vec<f32>`, `p0_threshold: f32`, `p1_min: f32`, `p1_max: f32`  
Output: `Vec<f32>`  
Logic:  
- P0: If `abs(signal[i]) < p0_threshold`, output 0.0 (negative space — avoid noise).  
- P1: If signal in `[p1_min, p1_max]`, pass through (safe channel).  
- P2: If signal outside P1 but above P0, apply scaling `signal * 0.5` (optimize within channel).  
- Strict priority: P0 > P1 > P2.

### **DeepSeek-chat implementation:**
```rust
/// Implements the Deadband Protocol for signal filtering.
/// P0: Zero out signals below threshold (negative space).
/// P1: Pass through signals within safe channel.
/// P2: Scale signals outside channel but above threshold.
pub fn deadband_filter(
    signal: Vec<f32>,
    p0_threshold: f32,
    p1_min: f32,
    p1_max: f32,
) -> Vec<f32> {
    if p1_min >= p1_max {
        panic!("Invalid P1 channel: min must be less than max");
    }
    
    signal.into_iter().map(|value| {
        let abs_val = value.abs();
        
        // P0: Map negative space
        if abs_val < p0_threshold {
            return 0.0;
        }
        
        // P1: Find safe channels
        if value >= p1_min && value <= p1_max {
            return value;
        }
        
        // P2: Optimize within channels
        value * 0.5
    }).collect()
}

#[cfg(test)]
mod tests {
    use super::*;
    
    #[test]
    fn test_deadband_priority() {
        let signal = vec![-0.5, 0.1, 1.0, 2.0, 3.0];
        let result = deadband_filter(signal, 0.2, 1.0, 2.0);
        // P0: 0.1 < 0.2 → 0.0
        // P1: 1.0 and 2.0 in channel → unchanged
        // P2: -0.5 and 3.0 outside channel → scaled
        assert_eq!(result, vec![-0.25, 0.0, 1.0, 2.0, 1.5]);
    }
}
```

### **Groq Llama-70b implementation:**
```rust
fn deadband_filter(signal: Vec<f32>, p0_threshold: f32, p1_min: f32, p1_max: f32) -> Vec<f32> {
    let mut output = Vec::with_capacity(signal.len());
    
    for &s in &signal {
        let abs_s = s.abs();
        
        // Deadband (P0)
        if abs_s < p0_threshold {
            output.push(0.0);
            continue;
        }
        
        // Safe zone (P1)
        if s >= p1_min && s <= p1_max {
            output.push(s);
            continue;
        }
        
        // Optimization zone (P2)
        output.push(s * 0.5);
    }
    
    output
}

// Example usage
fn main() {
    let data = vec![0.1, 0.5, 1.5, 2.5];
    let filtered = deadband_filter(data, 0.2, 1.0, 2.0);
    println!("{:?}", filtered); // [0.0, 0.25, 1.5, 1.25]
}
```

**Comparison:**
- **Correctness:** Both implement the priority correctly. DeepSeek includes input validation.
- **Code Quality:** DeepSeek version has doc comments, proper error handling, and tests. Groq version is minimal.
- **Fleet Context:** DeepSeek uses protocol terminology in comments; Groq uses generic terms.
- **Edge Cases:** DeepSeek checks `p1_min >= p1_max`; Groq doesn't.

## **3. Function 2: Tile Validator**

**Spec:**  
Input: `tile: serde_json::Value`  
Output: `Result<bool, String>`  
Checks:
1. Has required fields: `"agent"`, `"cycle"`, `"content"`, `"timestamp"`
2. `content` is non-empty string
3. `cycle` is positive integer
4. `timestamp` is ISO 8601 format
5. `agent` is one of known fleet roles

### **DeepSeek-chat implementation:**
```rust
use chrono::{DateTime, Utc};
use serde_json::Value;

const FLEET_AGENTS: [&str; 12] = [
    "navigator", "sentinel", "scribe", "tinker", "scout", "curator",
    "mason", "alchemist", "herald", "scholar", "weaver", "archivist"
];

/// Validates a training tile against fleet submission standards.
/// Returns Ok(true) if valid, Err with reason if invalid.
pub fn validate_tile(tile: &Value) -> Result<bool, String> {
    // Check required fields
    let required = ["agent", "cycle", "content", "timestamp"];
    for field in required.iter() {
        if !tile.get(*field).is_some() {
            return Err(format!("Missing required field: {}", field));
        }
    }
    
    // Validate agent
    let agent = tile["agent"].as_str().ok_or("Agent must be string")?;
    if !FLEET_AGENTS.contains(&agent.to_lowercase().as_str()) {
        return Err(format!("Unknown agent role: {}", agent));
    }
    
    // Validate cycle
    let cycle = tile["cycle"].as_i64().ok_or("Cycle must be integer")?;
    if cycle <= 0 {
        return Err(format!("Cycle must be positive, got {}", cycle));
    }
    
    // Validate content
    let content = tile["content"].as_str().ok_or("Content must be string")?;
    if content.trim().is_empty() {
        return Err("Content cannot be empty".to_string());
    }
    
    // Validate timestamp
    let timestamp = tile["timestamp"].as_str().ok_or("Timestamp must be string")?;
    DateTime::parse_from_rfc3339(timestamp)
        .map_err(|e| format!("Invalid ISO 8601 timestamp: {}", e))?;
    
    Ok(true)
}
```

### **Groq Llama-70b implementation:**
```rust
use serde_json::{Value, json};

fn validate_tile(tile: &Value) -> bool {
    // Check fields exist
    let has_fields = tile.get("agent").is_some() &&
                     tile.get("cycle").is_some() &&
                     tile.get("content").is_some() &&
                     tile.get("timestamp").is_some();
    if !has_fields { return false; }
    
    // Check types and values
    if !tile["agent"].is_string() { return false; }
    if !tile["cycle"].is_number() { return false; }
    if tile["cycle"].as_i64().unwrap_or(0) <= 0 { return false; }
    if !tile["content"].is_string() { return false; }
    if tile["content"].as_str().unwrap_or("").is_empty() { return false; }
    if !tile["timestamp"].is_string() { return false; }
    
    // Simple timestamp check (not full ISO 8601)
    let ts = tile["timestamp"].as_str().unwrap();
    if !ts.contains('T') || !ts.contains(':') { return false; }
    
    true
}
```

**Comparison:**
- **Correctness:** DeepSeek does full ISO 8601 validation; Groq does simple pattern check.
