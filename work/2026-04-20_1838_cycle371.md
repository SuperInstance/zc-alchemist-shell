## **2. Function 1: `deadband_filter`**

**Prompt to both models:**
```
Write a Python function named `deadband_filter` that implements the Deadband Protocol.

Inputs:
- `values`: A list of floating-point numbers.
- `threshold`: A non-negative float.

Logic:
- For each value in `values`:
    - If `-threshold <= value <= threshold`, set the output to 0.0.
    - If `value > threshold`, set the output to `value - threshold`.
    - If `value < -threshold`, set the output to `value + threshold`.

Return a new list with the processed values.

Include a brief docstring. Write only the function code.
```

### **DeepSeek-chat Output:**
```python
def deadband_filter(values, threshold):
    """
    Apply a deadband filter to a list of values.

    Parameters:
    values (list of float): Input values.
    threshold (float): Non-negative deadband threshold.

    Returns:
    list of float: Filtered values where values within [-threshold, threshold]
                   are set to 0, and others are adjusted by the threshold.
    """
    result = []
    for v in values:
        if -threshold <= v <= threshold:
            result.append(0.0)
        elif v > threshold:
            result.append(v - threshold)
        else:  # v < -threshold
            result.append(v + threshold)
    return result
```

### **Groq Llama-70b Output:**
```python
def deadband_filter(values, threshold):
    """
    Apply deadband filtering to a list of values.

    Args:
        values (list[float]): List of input values.
        threshold (float): Deadband threshold (non-negative).

    Returns:
        list[float]: List of filtered values.
    """
    filtered = []
    for val in values:
        if abs(val) <= threshold:
            filtered.append(0.0)
        elif val > threshold:
            filtered.append(val - threshold)
        else:  # val < -threshold
            filtered.append(val + threshold)
    return filtered
```

Both functions produce the correct output.

**Observations:**

*   **Style:** Both have clear docstrings. Groq's uses `Args` and `Returns` sections, which is a common standard. DeepSeek's uses `Parameters` and `Returns`.
*   **Variable Names:** Both are clear (`result` vs `filtered`).
*   **Edge Cases:** Both handle the inclusive deadband correctly. Both would handle an empty list.

## **3. Function 2: `parse_fleet_log`**

**Prompt to both models:**
```
Write a Python function named `parse_fleet_log` that parses a standardized fleet log line.

The line format is: "YYYY-MM-DD HH:MM:SS [SERVICE] LEVEL: message"
Example: "2026-04-20 18:34:15 [holodeck-rust] INFO: NPC sentiment updated"

Return a dictionary with the keys: `timestamp` (string), `service` (string, without brackets), `level` (string), `message` (string).

Assume the format is mostly correct, but be robust to extra spaces. If the line cannot be parsed, return a dict with all values set to None.

Include a brief docstring and error handling. Write only the function code.
```

### **DeepSeek-chat Output:**
```python
import re

def parse_fleet_log(line):
    """
    Parse a fleet log line into its components.

    Parameters:
    line (str): A log line in format "YYYY-MM-DD HH:MM:SS [SERVICE] LEVEL: message"

    Returns:
    dict: Dictionary with keys 'timestamp', 'service', 'level', 'message'.
          If parsing fails, all values are None.
    """
    pattern = r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) \[([^\]]+)\] (\w+): (.+)$'
    match = re.match(pattern, line.strip())
    if match:
        return {
            'timestamp': match.group(1),
            'service': match.group(2),
            'level': match.group(3),
            'message': match.group(4)
        }
    else:
        return {'timestamp': None, 'service': None, 'level': None, 'message': None}
```

### **Groq Llama-70b Output:**

```python
import re

def parse_fleet_log(log_line):
    """
    Parses a fleet log line.

    Expected format: "YYYY-MM-DD HH:MM:SS [SERVICE] LEVEL: message"

    Args:
        log_line (str): The log line to parse.

    Returns:
        dict: A dictionary with keys 'timestamp', 'service', 'level', 'message'.
              If parsing fails, all values are None.
    """
    # Regex to match the log format
    log_pattern = re.compile(
        r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) \[([^\]]+)\] (\w+): (.+)$'
    )

    match = log_pattern.match(log_line.strip())
    if match:
        return {
            'timestamp': match.group(1),
            'service': match.group(2),
            'level': match.group(3),
            'message': match.group(4)
        }
    else:
        return {
            'timestamp': None,
            'service': None,
            'level': None,
            'message': None
        }
```
