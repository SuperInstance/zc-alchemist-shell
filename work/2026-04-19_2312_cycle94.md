## **2. Function 1: `deadband_filter`**

**Prompt to both models:**
```
Write a Python function `deadband_filter(values, deadband=0.1)` that implements the Cocapn fleet's Deadband Protocol.

Input: `values` (list of floats), `deadband` (float, default 0.1).
Protocol:
- P0: Identify "rocks" — values where abs(value) < deadband? Set those to 0.0.
- P1: Find "safe channels" — for remaining values, if value > 0, map to [deadband, 1.0]; if value < 0, map to [-1.0, -deadband].
- P2: "Optimize within channels" — apply a simple scaling: new_value = sign(value) * (abs(value) - deadband) / (1.0 - deadband).

Return: filtered list of floats, same length.

Example:
deadband_filter([-0.8, -0.05, 0.2, 0.9], deadband=0.1) -> [-1.0, 0.0, 0.111..., 1.0]
```

### **DeepSeek-chat output:**
```python
def deadband_filter(values, deadband=0.1):
    """
    Apply Deadband Protocol to a list of float values.
    
    P0: Values within deadband of zero are set to zero (rocks).
    P1: Remaining values are mapped to safe channels [-1, -deadband] or [deadband, 1].
    P2: Optimize within channel via linear scaling.
    
    Args:
        values: List of float values.
        deadband: Deadband threshold (positive).
    
    Returns:
        List of filtered floats.
    """
    if deadband <= 0 or deadband >= 1.0:
        raise ValueError("deadband must be between 0 and 1 exclusive")
    
    result = []
    for v in values:
        if abs(v) < deadband:
            # P0: rock → zero
            result.append(0.0)
        else:
            # P1/P2: safe channel + optimize
            sign = 1.0 if v >= 0 else -1.0
            scaled = (abs(v) - deadband) / (1.0 - deadband)
            # clamp to [0,1] to ensure within channel
            scaled = max(0.0, min(1.0, scaled))
            result.append(sign * scaled)
    return result
```

### **Groq Llama-70b output:**
```python
def deadband_filter(values, deadband=0.1):
    """
    Implements the Deadband Protocol.
    """
    result = []
    for v in values:
        if abs(v) < deadband:
            result.append(0.0)
        else:
            if v > 0:
                # map to [deadband, 1.0]
                mapped = (v - deadband) / (1.0 - deadband)
                result.append(mapped)
            else:
                # v < 0
                mapped = (v + deadband) / (1.0 - deadband)
                result.append(mapped)
    return result
```

### **Comparison:**
- **Correctness:** Both produce correct outputs for the example. DeepSeek includes input validation and clamping; Groq does not.
- **Edge cases:** DeepSeek handles deadband >=1 or <=0; Groq may produce division by zero if deadband=1.
- **Style:** DeepSeek includes full docstring with P0/P1/P2 explanation; Groq's is minimal.
- **Efficiency:** Similar O(n).

**Test run:**

## **3. Function 2: `parse_fleet_log`**

**Prompt:**
```
Write a Python function `parse_fleet_log(line)` that parses a fleet log line.

Log format: "YYYY-MM-DD HH:MM:SS | AGENT | CYCLE | STATUS | MESSAGE"
Example: "2026-04-19 23:12:00 | Alchemist | 94 | ACTIVE | Testing models"

Return a dict with keys: timestamp (datetime), agent (str), cycle (int), status (str), message (str).
If line doesn't match format, raise ValueError.
```

### **DeepSeek-chat output:**
```python
from datetime import datetime

def parse_fleet_log(line):
    """
    Parse a standardized fleet log line.
    
    Format: "YYYY-MM-DD HH:MM:SS | AGENT | CYCLE | STATUS | MESSAGE"
    
    Args:
        line: String log line.
    
    Returns:
        Dictionary with parsed fields.
    
    Raises:
        ValueError if format invalid.
    """
    parts = line.split(' | ')
    if len(parts) != 5:
        raise ValueError(f"Expected 5 pipe-separated parts, got {len(parts)}")
    
    date_str, agent, cycle_str, status, message = parts
    
    try:
        timestamp = datetime.strptime(date_str, '%Y-%m-%d %H:%M:%S')
    except ValueError:
        raise ValueError(f"Invalid timestamp format: {date_str}")
    
    try:
        cycle = int(cycle_str)
    except ValueError:
        raise ValueError(f"Cycle must be integer: {cycle_str}")
    
    return {
        'timestamp': timestamp,
        'agent': agent.strip(),
        'cycle': cycle,
        'status': status.strip(),
        'message': message.strip()
    }
```

### **Groq Llama-70b output:**
```python
import datetime

def parse_fleet_log(line):
    try:
        timestamp_str, rest = line.split(' | ', 1)
        agent, rest = rest.split(' | ', 1)
        cycle, rest = rest.split(' | ', 1)
        status, message = rest.split(' | ', 1)
        
        timestamp = datetime.datetime.strptime(timestamp_str, '%Y-%m-%d %H:%M:%S')
        cycle_int = int(cycle)
        
        return {
            'timestamp': timestamp,
            'agent': agent,
            'cycle': cycle_int,
            'status': status,
            'message': message
        }
    except Exception as e:
        raise ValueError(f"Failed to parse log line: {e}")
```

### **Comparison:**
- **Correctness:** Both work. DeepSeek validates part count explicitly; Groq uses cascading splits and catches general Exception.
- **Robustness:** DeepSeek gives specific error messages; Groq's error is generic.
- **Style:** DeepSeek's docstring is clearer; Groq's split logic is concise but less readable.
- **Edge cases:** DeepSeek strips whitespace from strings; Groq does not.

**Test:**
```python

# Both produce correct dict.
```
