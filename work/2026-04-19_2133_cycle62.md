## **3. Function 1: deadband_filter**

**User Prompt:**  
```
Write a function `deadband_filter(data, lower, upper)` that implements the Deadband Protocol.
Input: `data` (list of floats), `lower` (float), `upper` (float).
Output: filtered list where values outside [lower, upper] are removed, but with hysteresis:
- Once a value exits the deadband, it must stay out until the trend reverses.
- If value goes above upper, keep it until it drops below lower.
- If value goes below lower, keep it until it rises above upper.
Return filtered list.
```

### **DeepSeek-chat Output:**
```python
def deadband_filter(data: list[float], lower: float, upper: float) -> list[float]:
    """
    Implements Deadband Protocol with hysteresis.
    
    P0: Validate inputs.
    P1: Apply deadband logic.
    P2: Return filtered list.
    
    Args:
        data: List of float values.
        lower: Lower deadband threshold.
        upper: Upper deadband threshold.
    
    Returns:
        Filtered list where values outside deadband are kept based on hysteresis.
    
    Fleet Note: This mimics sensor filtering in fleet-simulator.
    """
    if not data:
        return []
    
    if lower >= upper:
        raise ValueError("lower must be < upper")
    
    filtered = []
    # State: 0=inside, 1=above, -1=below
    state = 0
    
    for value in data:
        if state == 0:  # Currently inside deadband
            if value > upper:
                filtered.append(value)
                state = 1  # Now above
            elif value < lower:
                filtered.append(value)
                state = -1  # Now below
            else:
                # Inside deadband, discard
                pass
        elif state == 1:  # Currently above upper
            filtered.append(value)  # Keep all while above
            if value < lower:  # Crossed below lower
                state = -1
            # If value returns to deadband, stay in state=1
        elif state == -1:  # Currently below lower
            filtered.append(value)  # Keep all while below
            if value > upper:  # Crossed above upper
                state = 1
    
    return filtered
```

```python
from typing import List

def deadband_filter(data: List[float], lower: float, upper: float) -> List[float]:
    """
    Apply deadband filtering with hysteresis as per Fleet Deadband Protocol.
    
    Parameters:
    -----------
    data : List[float]
        Time-series data points
    lower : float
        Lower deadband threshold
    upper : float
        Upper deadband threshold
    
    Returns:
    --------
    List[float]
        Filtered data where values outside deadband are retained
        according to hysteresis rules.
    
    Hysteresis logic:
    - Once data leaves deadband (goes above upper or below lower),
      it remains "outside" until crossing the opposite threshold.
    - This prevents rapid toggling at boundaries.
    """
    if lower >= upper:
        raise ValueError("Deadband invalid: lower must be less than upper")
    
    if not data:
        return []
    
    filtered_data = []
    outside_state = None  # None = inside, 'above' = above upper, 'below' = below lower
    
    for val in data:
        if outside_state is None:
            # Currently inside deadband
            if val > upper:
                filtered_data.append(val)
                outside_state = 'above'
            elif val < lower:
                filtered_data.append(val)
                outside_state = 'below'
            # else: inside deadband, skip
        elif outside_state == 'above':
            # Currently above upper threshold
            filtered_data.append(val)  # Keep all values while above
            if val < lower:  # Crossed below lower threshold
                outside_state = 'below'
        elif outside_state == 'below':
            # Currently below lower threshold
            filtered_data.append(val)  # Keep all values while below
            if val > upper:  # Crossed above upper threshold
                outside_state = 'above'
    
    return filtered_data
```

- **Fleet context:** DeepSeek explicitly mentions fleet-simulator and uses fleet priority labels.
- **Error handling:** Both validate `lower < upper` and empty input.
- **Edge cases:** Both handle empty list. DeepSeek checks `if not data` first; Groq checks after validation.

**Verdict:** DeepSeek's code feels more fleet-native (mentions P0/P1/P2). Groq's is more generic but well-documented.

```python
import re
from datetime import datetime
from typing import List, Dict, Optional

def parse_fleet_log(log_text: str) -> List[Dict[str, str]]:
    """
    Parse fleet service logs into structured records.
    
    P0: Don't crash on malformed lines.
    P1: Extract timestamp, level, agent, message.
    P2: Use efficient parsing.
    
    Fleet Note: Used by sentinel agents for health monitoring.
    """
    lines = log_text.strip().split('\n')
    records = []
    
    # Regex matches: 2026-04-19 21:30:45 [INFO] oracle1: Heartbeat OK
    log_pattern = r'^(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}) \[(\w+)\] (\w+): (.+)$'
    
    for line in lines:
        line = line.strip()
        if not line:
            continue
            
        match = re.match(log_pattern, line)
        if match:
            timestamp, level, agent, message = match.groups()
            # Validate timestamp format
            try:
                datetime.strptime(timestamp, '%Y-%m-%d %H:%M:%S')
            except ValueError:
                # Malformed timestamp, skip line
                continue
                
            records.append({
                'time': timestamp,
                'level': level,
                'agent': agent,
                'message': message
            })
        else:
            # P0: Skip malformed lines silently
            # Could log to debug in real implementation
            pass
    
    return records
```
