## **2. Function 1: `parse_log_line`**

**Prompt:**
```
Write a Python function called `parse_log_line` that takes a single string argument `line`.
The log line format is: `[TIMESTAMP] LEVEL: message | key1=value1 key2=value2`
Example: `[2026-04-20T01:28:00Z] INFO: Agent cycle started | agent=Alchemist cycle=148`
The function must return a dictionary with keys: `timestamp`, `level`, `message`, and `extras`.
`extras` should be a dictionary of the key=value pairs. If there are no extras, it should be an empty dict.
Handle malformed lines gracefully (return None).
Include type hints and a docstring.
```

### **DeepSeek-chat Output:**
```python
import re
from datetime import datetime
from typing import Optional, Dict, Any

def parse_log_line(line: str) -> Optional[Dict[str, Any]]:
    """
    Parse a structured log line into a dictionary.

    Format: [TIMESTAMP] LEVEL: message | key1=value1 key2=value2

    Args:
        line: A log line string.

    Returns:
        A dictionary with keys: 'timestamp', 'level', 'message', 'extras'.
        Returns None if the line is malformed.
    """
    # Main regex to capture timestamp, level, message, and optional extras
    pattern = r'^\[([^\]]+)\]\s+([A-Z]+):\s+(.+?)(?:\s+\|\s+(.+))?$'
    match = re.match(pattern, line.strip())
    if not match:
        return None

    timestamp_str, level, message, extras_str = match.groups()

    # Parse timestamp (try ISO format, fallback to string)
    try:
        timestamp = datetime.fromisoformat(timestamp_str.replace('Z', '+00:00'))
    except ValueError:
        timestamp = timestamp_str  # Keep as string if parsing fails

    # Parse extras if present
    extras = {}
    if extras_str:
        # Split by spaces, then by '='
        for pair in extras_str.split():
            if '=' in pair:
                key, value = pair.split('=', 1)
                extras[key] = value

    return {
        'timestamp': timestamp,
        'level': level,
        'message': message,
        'extras': extras
    }
```

### **Groq Llama-3.3-70b-Specdec Output:**

```python
    """
    Parses a log line in the format:
    [TIMESTAMP] LEVEL: message | key1=value1 key2=value2

    Returns a dict with keys: timestamp, level, message, extras.
    Returns None if the line is malformed.
    """
    line = line.strip()
    # Regex to capture the four parts
    match = re.match(r'^\[([^\]]+)\]\s+(\w+):\s+(.+?)(?:\s+\|\s+(.+))?$', line)
    if not match:
        return None

    timestamp, level, message, extras_str = match.groups()

    # Parse extras
    extras: Dict[str, str] = {}
    if extras_str:
        # Split by spaces, each is key=value
        for item in extras_str.split():
            if '=' in item:
                key, val = item.split('=', 1)
                extras[key] = val

    return {
        'timestamp': timestamp,  # kept as string per spec
        'level': level,
        'message': message,
        'extras': extras
    }
```

### **Comparison for Function 1:**
- **Correctness:** Both handle the core format. DeepSeek attempts to parse the timestamp to a `datetime` object (beyond spec, but useful). Groq keeps it as a string (strict to spec). Both handle missing extras.
- **Clarity:** Both have clear docstrings and comments. DeepSeek's regex is slightly more explicit (`[A-Z]+` vs `\w+` for LEVEL). Groq's regex uses `\w+` which could allow lowercase (potential bug).
- **Efficiency:** Similar regex patterns, comparable.
- **Spec Compliance:** Groq is more literal. DeepSeek adds a feature (datetime conversion). Both meet the core requirement.
- **Edge Cases:** DeepSeek's timestamp fallback is robust. Groq's `\w+` might be too permissive for log levels.

**Verdict:** DeepSeek's version is more robust and feature-rich. Groq's is adequate but has a minor regex issue.

## **3. Function 2: `find_safe_channels`**

**Prompt:**
```
Write a Python function called `find_safe_channels` that implements the P1 step of the Deadband Protocol.
It takes two arguments: `negative_space` (list of coordinates to avoid) and `search_bounds` (tuple (x_min, x_max, y_min, y_max)).
The function must return a list of "safe channels". A channel is defined as a continuous vertical strip (constant x) where no point (x, y) within the bounds is in the negative_space.
Assume coordinates are integers. Be efficient for large spaces.
Include type hints, a docstring, and a brief example in the docstring.
```

### **DeepSeek-chat Output:**
```python
from typing import List, Tuple, Set

def find_safe_channels(negative_space: List[Tuple[int, int]], 
                       search_bounds: Tuple[int, int, int, int]) -> List[int]:
    """
    Find safe vertical channels (x-coordinates) within bounds that avoid negative space.

    P1 of Deadband Protocol: Find safe channels (where you CAN be).

    Args:
        negative_space: List of (x, y) coordinates to avoid.
        search_bounds: (x_min, x_max, y_min, y_max) defining rectangular search area.

    Returns:
        List of x-coordinates representing safe vertical channels.

    Example:
        >>> find_safe_channels([(1, 5), (2, 3)], (0, 5, 0, 10))
        [0, 3, 4, 5]  # x=1 and x=2 are blocked
    """
    x_min, x_max, y_min, y_max = search_bounds
    
    # Convert negative space to a set for O(1) lookups
    blocked_set = set(negative_space)
    
    # Determine which x-values have any blocked point within the vertical range
    blocked_x = set()
    for x, y in blocked_set:
        if x_min <= x <= x_max and y_min <= y <= y_max:
            blocked_x.add(x)
    
    # Safe channels are all x in range that are not blocked
    safe_channels = [x for x in range(x_min, x_max + 1) if x not in blocked_x]
    return safe_channels
```
